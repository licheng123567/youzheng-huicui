# 有证慧催 PRD · M6 存证

> 依赖：[00-总纲](./00-总纲.md) §1 护城河②、§2 计费、[M4 催收作业](./04-催收作业.md)（法务文书/通话录音是输入）。
> 目标：定义**区块链存证**三类业务场景、决定权、计费（只向物业）、与法务/诉讼的衔接。
> **契约对齐**：本模块与 API 契约 v1.0.0 对齐。

---

## 1. 模块范围

- **存证三场景**：①**法务文书送达存证**（签收拍照等）；②**通话录音存证**（**由协调员按需决定、无法务前置**，一般录音不强制存证）；③**证据材料打包存证**（协调员打包下载催收+沟通记录时，决定是否存证）。**线下回款转账凭证不存证**。
- 存证的**计费（只向物业收费）**、开关/通道 Key 配置。
- 存证记录的归属、可追溯、不可篡改，**为后期诉讼提供法律证据**。

**不在本模块**：法务文书出具/送达动作本身（M4）、录音解析（M5）、结算明细（M9）。

---

## 2. 关键概念

| 概念 | 说明 |
|---|---|
| 存证 | 关键证据上链留痕、不可篡改，为诉讼提供法律证据 |
| 送达存证（场景①） | 法务文书（催收函/律师函/起诉状等）线下送达的签收拍照等留痕上链 |
| 通话录音存证（场景②） | **一般录音不强制存证**；**由协调员按需决定**是否对关键录音存证（线下判断、系统执行，**无法务前置**），作为后续法务/举证依据 |
| 证据材料打包存证（场景③） | 法务处理中，协调员把催收记录+沟通记录打包下载为证据材料时，决定该次打包是否上链存证 |
| 存证凭证 | 上链后返回的存证编号/哈希/时间戳等，可供核验 |

---

## 3. 业务规则

- **BR-M6-01（三类场景）**：存证仅三类——①**法务文书送达存证**（M4 协调员送达签收拍照后生成）；②**通话录音存证**（**由协调员按需决定，无法务前置**，见 BR-M6-02）；③**证据材料打包存证**：协调员把**催收记录 + 沟通记录打包下载为证据材料**时，由协调员决定该次打包是否存证。**线下回款转账凭证不存证**。
- **BR-M6-02（录音存证 = 协调员按需决定，无法务前置）**：录音存证**由协调员按需决定**、随时可对关键录音存证并在系统内执行，**不以"申请律师函/发起法务"为前置**——**律师函与催收单一样，是按需付费的送达文书，与录音存证无依赖关系**。**一般录音不强制存证**，需要作证据时协调员决定即可（是否值得存证为线下判断，无"平台法律服务团队/律师"系统角色）。送达存证、证据材料打包存证同样由协调员在对应动作后按需发起。
- **BR-M6-08a（各场景存证打包内容明细）**：三类存证各按 **ZIP** 打包，内容明细——**①通话录音存证**：录音文件 + ASR 转写文本 + 通话人 + 通话时间 + 通话时长；**②工单存证**（送达/证据材料下的工单类）：工单内容 + 工单图片（若有）+ 处理人 + 处理时间；**③整案打包存证**：本案**全部沟通记录文本** + **相关文件**（催收单 / 律师函等送达文书存证文件）+ **已生成的存证证书**。上链仍只存哈希/元数据、原件加密存平台（BR-M6-04a）；每次打包存证生成可下载、可独立核验的证书（BR-M6-04）。
- **BR-M6-03（只向物业收费）**：存证**只向物业公司收费**（CFG-EVIDENCE-PRICE），不向服务商收；**按次计费、三类场景同一单价**（单价数值 TBD）；存证能力开关/通道 Key 由平台超管配置（CFG-EVIDENCE）。
- **BR-M6-04（不可篡改 + 可核验 + 可下载证书 + 第三方核验）**：存证上链后**不可篡改**，返回**存证凭证**（编号/哈希/时间戳）；**每次存证均生成存证证书，有权角色可下载**（PDF 等，格式 TBD）；任何有权角色可**核验**。**证书带可公开独立核验的入口（如核验链接/二维码），支持第三方/法院凭证书自助核验**（无需登录平台）。适用全部三类存证场景。
- **BR-M6-04a（存证内容范围）**：**哈希/元数据上链 + 原件加密存平台**——上链仅存文件哈希与元数据（时间戳等），**原件不上链、加密存平台**；举证时以"原件 + 链上哈希比对"自证未篡改，避免业主隐私数据上公链。
- **BR-M6-05（归属与追溯）**：存证记录归属案件，随案保留（BR-M4-10）；**结案/归档后仍可供平台/物业审计与诉讼追溯**（M8 BR-M8-10）。
- **BR-M6-06（失败处理）**：上链失败须可见原因、可重试；失败不计费（口径 TBD，OQ-M6-2）。
- **BR-M6-07（隔离与脱敏）**：存证内容含敏感信息，访问受 M1 隔离；结案后服务商不可见明细（M8 BR-M8-09），但平台/物业可调取作证据。
- **BR-M6-08（证据材料包结构）**：证据材料包为 **ZIP**，含——**①催收时间线 PDF（表格形式：时间/动作/操作人/结果等）**；**②过程文件**（录音文件、转写 text、工单、送达记录、法务文书等）。打包时协调员可选是否对该包存证（BR-M6-01 场景③）。包内字段/文件清单细节 TBD。

## 4. Epic 与用户故事

### Epic M6-A：送达存证（场景①）

**US-M6-01 法务文书送达存证**
- 作为**物业协调员**，我想对法务文书送达做存证（签收拍照上链），以便诉讼时有送达证据。
- 验收标准：
  - **Given** 我已线下送达某法务文书并签收拍照（M4 US-M4-09），**When** 我发起送达存证，**Then** 签收照片+送达记录上链，返回存证凭证并关联案件（BR-M6-01/04）。
  - **Given** 存证完成，**When** 查看案件，**Then** 可见该存证凭证、**可下载存证证书**并可核验（BR-M6-04）。
  - **Given** 上链失败，**When** 重试，**Then** 成功后返回凭证；失败不计费（BR-M6-06）。

### Epic M6-B：通话录音存证（场景②，协调员按需决定）

**US-M6-02 协调员按需对录音存证**
- 作为**物业协调员**，我想对关键录音随时决定是否存证（无需先申请律师函/发起法务），以便支撑后续法务或举证。
- 验收标准：
  - **Given** 某关键录音需作证据，**When** 我（协调员）对该录音执行存证，**Then** 上链返回凭证、可下载证书并关联案件——**无"申请律师函/发起法务"前置**（BR-M6-01/02/04）。
  - **Given** 录音存证发起，**When** 打包，**Then** 包内含 录音文件 + ASR 转写文本 + 通话人 + 通话时间 + 通话时长（BR-M6-08a）。
  - **Given** 一般录音协调员未选存证，**When** 系统处理，**Then** **不强制存证**（仅常规留存）（BR-M6-02）。

### Epic M6-B2：证据材料打包存证（场景③）

**US-M6-05 证据材料打包下载与存证**
- 作为**物业协调员**，我想在法务处理中把催收记录+沟通记录打包下载为证据材料，并决定是否对该打包存证，以便诉讼举证。
- 验收标准：
  - **Given** 案件在法务处理中，**When** 我打包下载证据材料，**Then** 系统生成 **ZIP**：含**催收时间线 PDF（表格形式）** + 过程文件（**录音文件、转写 text、工单等**）（BR-M6-08）。
  - **Given** 我打包时选择存证，**When** 确认，**Then** 该证据包上链返回存证凭证、关联案件（BR-M6-01/04）。
  - **Given** 我打包时未选存证，**When** 下载，**Then** 仅下载、不上链。

### Epic M6-C：计费与核验

**US-M6-03 存证计费（只向物业）**
- 作为**平台**，我想对存证只向物业计费，以便符合商业模式。
- 验收标准：
  - **Given** 一次存证成功，**When** 计费，**Then** **按次**记物业侧费用（三类场景同一单价，CFG-EVIDENCE-PRICE），不向服务商计费（BR-M6-03）。
  - **Given** 服务商查看案件，**When** 涉及存证费用，**Then** 不可见物业存证计费（隔离，M1）。

**US-M6-04 存证核验**
- 作为**有权角色（平台/物业）**，我想核验存证真实性，以便作为诉讼证据。
- 验收标准：
  - **Given** 案件有存证凭证，**When** 我核验，**Then** 系统校验上链记录一致性并显示存证时间/哈希等（BR-M6-04）。
  - **Given** 案件已结案/归档，**When** 平台/物业调取存证，**Then** 仍可核验追溯（BR-M6-05）。
  - **Given** 第三方/法院持有存证证书，**When** 通过证书上的核验入口/二维码核验，**Then** 无需登录平台即可独立核验真伪（BR-M6-04）。
