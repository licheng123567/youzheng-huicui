# 有证慧催 PRD · M4 催收作业

> 依赖：[00-总纲](./00-总纲.md)、[M3 流转](./03-案件流转与公海.md)（跟进记录是释放计时 T_collector 的输入）。
> 目标：定义催收作业动作——外呼通话+录音（**任一关联角色均可打**）、标注跟进、承诺、转工单、建议法务、发缴费链接，以及协调员的线下回款/法务主导（平台出具+协调员送达签收拍照+存证）。
> **契约对齐**：本模块与 API 契约 v1.0.0 对齐。

---

## 1. 模块范围

- 给案件联系人外呼通话与自动录音（催收员对私海案件、物业负责人/协调员对关联案件均可，App/PC）。
- 通话解析分钟数消耗与余额不足暂停解析（CFG-MIN-STOP，不影响打电话/录音）。
- 跟进记录：通话结果标注、小结、承诺还款（标注类动作仅持有催收员）。
- 发缴费链接（业主 H5 入口，金额=减免后应收）。
- 转工单要求；催收员"建议法务"轻量标注（不交接案件）。
- 物业协调员：标记线下回款、处理工单回执、**独立主导法务（催收函/律师函/要素化起诉状→平台出具/打印→送达签收拍照→存证）**、催费单链接。

**不在本模块**：录音转写/AI 话术（M5）、存证细节（M6）、缴费 H5 内部（M7）、结算（M9）、结案判定（M8）。

---

## 2. 关键概念

| 概念 | 说明 |
|---|---|
| 跟进记录 | 通话 / 标注 / 小结 / 承诺 等任一动作；是 T_collector 释放计时的重置依据（M3 BR-M3-08） |
| 通话录音 | 外呼通话自动录音，作为转写计费（M5）与存证（M6）的来源 |
| 承诺还款 | 业主承诺的还款金额/时间，形成待跟踪项 |
| 缴费链接（=催费单链接） | 发给业主的 H5 链接，展示应缴款信息+物业收款渠道，金额=减免后应收（M2 BR-M2-05）；本期线下缴、协调员标注，在线缴费后期（BR-M4-19） |
| 转工单 | 催收员请求物业协调员线下协调（如上门、开证明） |
| 建议法务（轻量标注） | 催收员对某案打的"建议走法务"标注，提醒协调员关注；**不改变案件归属、不移出私海** |
| 申请律师函 / 法律文书（协调员主导 · 按需付费） | 协调员**按需向平台申请**律师函 / 催收函 / 要素化起诉状（**入口即"申请律师函"**；**法律服务按次付费**，M9）→ 平台出具/打印 → 线下送达 → 签收拍照 → 存证；**与催收并行**。（原"发起法务"为此动作的口径细化——其本质即向平台按需申请付费法律文书） |

## 3. 业务规则

- **BR-M4-01（催收作业权限）**：**催收动作（标注/承诺/转工单/建议法务/释放）仅持有该私海案件的催收员**可执行；非持有人/非本服务商不可操作（隔离，M1）。
- **BR-M4-01a（通话权限，任一角色）**：**给案件联系人打 App 电话**不限于催收员——**持有案件的催收员、案件关联的物业负责人/协调员**均可打；每次通话**自动录音 + 转写**并**关联为该案件跟进记录**，案件关联人（有权者）可回放/看文本/取策略（承接总纲 §6、BR-M4-10）。
- **BR-M4-01b（通话与录音获取模型：系统不主动拨号·本机录音自动上传）**：平台**不主动发起外呼、不感知拨打时机**。
  - **流程**：App 内**点击案件电话号码 → 跳起手机系统拨号器本机通话**；**通话结束、系统检测到通话完成 → App 自动检测本机系统录音目录是否产生该通录音 → 有则自动上传云端解析（ASR）**、关联到本案件并生成通话跟进记录（含时长，重置 T_collector BR-M4-03）。
  - **判断有无录音**：App 须**判断本次通话是否有录音文件生成**（机型/权限差异可能无文件）；**无文件 → 提示手动上传**。**App 未自动上传 / 无文件时，支持手动选择录音文件上传**，走同一条解析链路（M5）。
  - **「获取最新通话录音」= 查最近一通的录音/解析状态**（不是通话记录翻阅）：展示最近一次通话**有没有录音上来、解析到哪一步**（无录音 / 上传中 / 解析中 / 已就绪），就绪后直达"查看并标注（AI 复盘）"，无录音则引导手动上传。
  - **服务端契约边界**：服务端只负责**接收上传 + ASR 解析 + 状态查询**；"读本机录音目录/拉最近一通"是 **App 客户端本地行为**，不是服务端接口。
  - **BR-M4-01c（首次设置 + 录音目录可改 + 自动录音引导）**：App **首次登录引导**完成 ①麦克风/通话录音权限、②存储/文件访问权限、③**录音目录检测**、④开通**系统通话自动录音**。因**机型差异**录音目录路径不同，App 提供**常见机型预设（小米/华为/OPPO·一加/vivo/通用）+ 手动修改路径 + 重新检测**；检测不到录音时，可调整目录或转 Web 手动上传（救济）。开通自动录音的分机型指引随附。
- **BR-M4-02（通话录音 vs 语音解析）**：**打电话 + 录音始终可用，不受分钟余额限制**。**通话解析分钟数消耗的是"录音上传平台做语音解析（STT 转文字 + 给策略）"环节**，按发起方所属组织（服务商或物业）计费扣减（M5/M9）。
- **BR-M4-02a（余额不足只停解析，不停通话）**：
  - 余额 **≤ CFG-MIN-WARN（如 10 分钟）** → **预警提醒充值**；
  - 余额不足 / **≤ CFG-MIN-STOP** → **暂停自动语音解析**（通话照常进行、录音照常保存，只是不自动转写/给策略）；
  - **充值成功后**，对未解析的录音**可手动触发解析**补转（M5）。
- **BR-M4-03（跟进记录与 T_collector）**：
  - **有效跟进**（重置 T_collector）= 催收员对其私海案件的**主动催收动作**：**接通通话** / 接通类标注 / 小结 / 承诺 / 登记承诺 / **建议法务**等。
  - **未接通的外呼**（未接 / 拒接 / 空号 / 关机等"未接通类"标注）**仅留痕、不算有效跟进、不重置 T_collector**（避免空拨占案）。
  - **物业负责人/协调员的通话**生成案件跟进记录（进时间线、可见），但**不驱动池流转、不重置 T_collector**（M3 BR-M3-08）。
- **BR-M4-03a（写跟进记录 ≠ 通话结果标记）**：两类动作区分——**写跟进记录**：随时写的**自由手记**（跟进方式 + 内容，**可附图片 / 音频 / 文件等附件**），写入案件时间线，**不改变案件状态**；**通话结果标记**：对某通已解析通话打**结果标签**（接通/未接/拒接/承诺/失联等，CFG-MARK-CODES），在**解析后 AI 复盘**中完成，按"是否有效跟进"影响 T_collector（BR-M4-03、M5 BR-M5-04a）。二者均进时间线，均不等同于状态变更（状态变更走承诺/回款/撤案等专门动作）。
- **BR-M4-13（承诺还款管理）**：承诺支持**单笔**或**分期**（每期金额 + 日期）。系统对每期**到期前预警 + 到期提醒**（提前量**复用 CFG-RELEASE-WARN**），**承诺待办展示在催收员工作台显著位置**。系统将**承诺与实际回款对照**，逐期标记**已兑现 / 部分兑现 / 违约**，沉淀为该业主/案件的履约记录，供后续催收策略与信用参考。承诺及其兑现状态归属案件（BR-M4-10）。**承诺分期 ≠ 政策分期：承诺仅为履约跟踪，不改变案件应收/结清依据**（M2 BR-M2-16）。
  - **提醒渠道**：到期前/到期提醒经**催收员工作台待办 + App 推送**（不向催收员发短信，省成本）。
  - **违约升级**：某期承诺到期未回款 → 标记**违约**、计入催收员/案件违约记录，并在**服务商负责人团队看板**体现（不单独强推，负责人自管）。
- **BR-M4-12（标注枚举可配）**：通话结果标注为受控枚举（**枚举集待定 TBD**），**由平台超管统一配置**（全局，CFG-MARK-CODES）；每个枚举项各含两个布尔标记——**connected（是否接通）** 与 **effectiveFollowUp（是否有效跟进）**；effectiveFollowUp=true 时服务端重置 T_collector（BR-M4-03）。
- **BR-M4-04**：发缴费链接的金额取**减免后应收**（含分期则取当期应收）；链接指向业主 H5（M7）。
- **BR-M4-19（缴费链接 = 催费单链接；本期线下、在线后期）**："缴费链接"与"催费单链接"**为同一事物**。**本期**：链接（H5）向业主**展示应缴款信息 + 物业收款渠道**（收款账户/通道取自项目"收款信息"，M2），业主据此**线下缴费**，再由协调员**标注回款**（US-M4-08）。**在线缴费（业主在线支付、实时到账、系统自动结清、免协调员标注）为后期延伸**（M7）。
- **BR-M4-14（缴费链接生命周期）**：缴费链接**带有效期**（时长可配 TBD），支持**重发**与**作废**；过期/作废链接不可用于缴费，业主访问时提示失效；发送/重发/作废均留痕（归属案件）。
- **BR-M4-14a（防滥用）**：**同一案件短信发送有冷却间隔**（如 X 分钟内不可重复发短信，阈值可配 CFG-SMS-COOLDOWN）；当前有效链接未过期时优先提示**重发**而非新发。**复制/转微信不限频**（不耗条数、无成本）。**不设催收员单日发送硬上限**——由物业**短信条数余额**天然约束（余额耗尽即发不出）。
- **BR-M4-15（链接送达方式）**：缴费链接可经**短信**送达，或由发起人**复制/转发到微信**等渠道分享。**短信送达走平台统一短信通道**（平台配置的签名+模板），按条扣减**该物业**的短信条数；**复制/转发到微信不走短信、不扣条数**。
- **BR-M4-16（短信通道）**：**短信签名与模板由平台按项目统一配置/提供（不同小区可用不同签名），物业不创建、无审核流程**。物业只需：**充值短信条数**（平台给物业充值、按条计费、低于 CFG-SMS-WARN 预警）、**使用平台配好的签名+模板发缴费短信**、**查看发送记录/情况/明细（统计+明细合并、按时段筛选、可导出，BR-M9-08）**。配置/充值/明细详见 **M9**。
- **BR-M4-05**：催收员**不能结案**（M8 BR）；可标注跟进、转工单、**打"建议法务"轻量标注**、主动释放回公海（M3）。**对无价值案件只有"主动释放"或"不动待 T_collector 自动释放"两条路，不设"转法务"硬交接动作**。
- **BR-M4-05a（建议法务轻量标注）**：催收员可对某案打"建议法务"标注，**提醒协调员关注**；该标注**不改变案件归属、不移出私海、不改变案件状态**，仅为信号；属催收员本人的有效跟进记录，**重置 T_collector**（BR-M4-03）。
- **BR-M4-06**：转工单后由**物业协调员**接续处理并回执（BR-M4-17）；任何转出/标注**不因此擅自结案**。
- **BR-M4-18（协调员按需申请律师函/法律文书，与催收并行）**：物业协调员对其**关联范围内任意案件**（无论在公海/私海/由谁持有）**可按需申请律师函 / 法律文书**——催收函 / 律师函 / 要素化起诉状（**操作入口命名为"申请律师函"，本质=向平台按需申请付费法律文书，法律服务按次付费 M9**）：选模板 → **平台用要素化数据套模板自动生成 PDF**（可打印）→ 线下送达 → **签收拍照** → **存证**（M6）。法务track与催收track**并行**，**不把案件挤出催收员私海**、不改变其池归属。法务动作与文书归属案件、留痕（BR-M4-10）。
- **BR-M4-18a（要素化诉状生成与预览）**：生成「要素化起诉状」时，平台用**项目档案（原告/合同/收费/违约金/坐落）+ 案件明细（被告/欠费）+ 催收过程（M4/M6 证据）**三方数据**套[要素化诉状模板](../prd/要素化诉状模板.md)**自动出具 PDF。生成前提供**要素预览**：自动填充已有要素、**缺失的必要要素标红提示**；缺失时可先到**项目档案 / 案件**补录（**诉讼要素允许后补**，BR-M2-15），补齐后再出具正式 PDF。证据材料按 ZIP 打包（BR-M6-08）。
- **BR-M4-17（工单）**：转工单含**类型枚举**（平台默认：**上门核实 / 材料证明 / 法务工单 / 其他**；**后期可配**）；工单转出后**案件仍留在催收员私海、催收不中断**；协调员处理完**回执给催收员**（结果+附件），催收员据此继续催收。工单全过程留痕、归属案件。
- **BR-M4-07（线下回款）**：协调员标记的**线下回款**计入案件回款，参与 M8 结清判定与 M2 聚合。
  - 可**上传凭证**（转账截图/收据，**非必选**）。（回款凭证本身不单独存证；法务举证通过证据材料打包存证，M6 场景③）
  - 登记后**通知持有催收员**（避免其继续催已缴费业主）。
  - 使**减免后应收全部到账** → 触发 **M8 自动结清结案**（"已结清"）；**部分到账（含分期某期）→ 记部分回款、案件仍"进行中"**，全部期次到账才结清（M8 BR-M8-01）。
  - **误登记可冲正/撤销，须留痕**（操作人/时间/原因）；冲正后回款额与案件状态相应回退（若已因该笔结清则回退为进行中，见 M8 OQ-M8-5）。
- **BR-M4-08**：所有催收动作留痕（操作人/时间/案件），录音留存供质检（M5）与存证（M6）。
- **BR-M4-09**：App 与 PC 的跟进记录互通，同一案件视图一致。
- **BR-M4-11（多联系方式管理）**：案件可含多个联系电话，每个号码支持：**标签**（平台默认：**本人 / 亲属 / 朋友 / 其他**；**后期可配**）、**设为主号码**（默认优先拨打）、**标注无效/空号**（号码列表置灰、默认不作为联系/拨打对象）。号码的增改与状态变更留痕，归属案件（BR-M4-10）。
- **BR-M4-20（催收员工作台聚合）**：催收员工作台**聚合其待办与提醒**——承诺到期（含临近预警）、案件释放预警、工单回执、被分配/抢到的新案件、建议法务跟进等，**按紧急度排序**（承诺/释放预警优先）。承接全局"提醒/待办聚合"（总纲 §9）。具体布局原型阶段细化。
- **BR-M4-20a（一线办案角色工作台 = 今日驾驶舱 · 原型阶段细化）**：**一线办案角色（催收员 CO / 物业协调员 PC）**的工作台采用**"今日驾驶舱"（今日必办 master-detail）**形态——左侧"今日必办"待办列表（**紧急度左侧色条 + 标签过滤 + 顶部 KPI 可点即筛**），点某案右侧**直接进统一三栏 CRM 工作台**；办完一条**自动跳下一条**（流式办案）；未选案件时右侧显示**"今日概览"**（重点提醒 + 今日进度）。**待办分类按角色**：CO = 承诺到期 / 临近自动释放 / 工单待回执 / 新分配；PC = 法务·待送达 / 工单待回执 / 线下回款待标 / 缴费链接待发。**管理/盯盘角色（物业负责人 PL / 平台超管 SA / 平台员工 SE / 服务商负责人 VL）工作台保持仪表盘形态**（KPI + 待办入口 + 趋势），不强加驾驶舱。
- **BR-M4-22（通话记录查询）**：提供独立「通话记录」入口，**汇总本人/可见范围内已产生的通话**（M4 通话跟进记录 + M5 解析结果），支持**按业主姓名 / 房号 / 电话搜索**，并可**按项目 / 批次 / 时间区间过滤**；每条可直达**该通 AI 复盘（M5 BR-M5-04a）**或进入案件。数据**按角色数据范围裁剪**（催收员=本服务商、协调员=本物业，等）；**结案后按 M8 BR-M8-09 对服务商/催收员收敛**。
- **BR-M4-23（催收员↔协调员处理结果互推闭环）**：催收员与协调员之间的处理结果**双向推送**形成闭环——催收员**转工单（BR-M4-17）**→ 协调员收到"待处理工单"提醒；协调员**工单回执 / 标记线下回款 / 律师函进展**等 → **持有催收员**收到回执提醒。推送落到对方**消息中心（未读提示）**并在**案件时间线**留痕；催收员在「消息中心」+ 案件三栏「沟通记录」+ 工作台待办 **三处同步可见**，确保线下处置结果及时回流、不脱节。
- **BR-M4-24（案件级协作 · 协调员/催收员双视角 · 原型阶段细化）**：案件三栏 CRM 工作台在**同一案件维度**提供协作视图，**数据同源**（同一组协作项：工单[类型+诉求+转出人] / 建议法务 / 减免申请 / 线下回款待核），按角色呈现两个视角：
  - **协调员视角「待办事项」**（协调员打开案件**默认进入**）：聚合**本案需其处理的协作项**，逐项给出诉求说明与「**处理并回执**」操作；处理后状态置「已处理」并记录结果/处理人/时间。
  - **催收员视角「协调员处理」**：展示**本案转协调员处理的事项及处理结果**（状态：待协调员处理 / 协调员已处理 + 结果摘要 + 处理人时间）。
  - 两视角**实时联动**：协调员处理 → 催收员侧状态/结果同步更新，并按 **BR-M4-23** 自动回执（消息中心 + 案件时间线 + 工作台待办）。
  - 本规则是 BR-M4-17（工单回执）/ BR-M4-23（互推闭环）在**单案维度的可视化落地**，使"本案协调员↔催收员协作"在同一案件内**双向闭环**、诉求可见、处置可追。
- **BR-M4-21（同业主多案件提示，避免重复触达）**：案件详情**提示"同一业主在其他批次的在催案件"**，避免重复触达。**受数据隔离约束**：催收员仅提示**本服务商可见范围内**的同业主案件；平台可见全部；**不跨服务商泄露他商案件明细**。同业主识别口径 = **姓名 + 身份证号**（身份证号属要素化起诉字段，缺失则不参与匹配）。
- **BR-M4-10（催收过程数据归属案件）**：所有催收过程数据——跟进记录、通话录音、转写文本、承诺、待办、工单/法务记录等——**归属"案件"而非催收员**。案件释放回公海/再分配/被新催收员抢单后，**历史数据全部随案保留**，新持有催收员可查看完整催收历史（含此前录音回放与文本）。**结案后**服务商/催收员对该案收敛为"只看统计、不看明细"（BR-M1-06 / M8 BR-M8-09）。

## 4. Epic 与用户故事

### Epic M4-A：外呼通话与录音

**US-M4-01 本机通话与录音回填**
- 作为**催收员**，我想用本机给案件联系电话通话、通话后把录音回填到案件并自动解析，以便高效催收且留痕。
- 验收标准：
  - **Given** 我持有某私海案件，**When** 我在 App 发起通话（跳起本机拨号器）或自行拨打该案号码，**Then** 通话在本机进行、由系统录音（BR-M4-01b；系统不主动拨号、不感知拨打时机）。
  - **Given** 我在 App 点案件号码本机通话、通话结束，**When** App 检测到系统录音文件并自动上传，**Then** 生成一条通话跟进记录（含时长）并重置该案件 T_collector（BR-M4-03）；本服务商有余额时**自动解析**、消耗分钟数（BR-M4-02）。点**"获取最新通话录音"**可查最近一通是否有录音上来及解析状态（无录音/上传中/解析中/已就绪 BR-M4-01b）。
  - **Given** 本次通话无录音文件 / App 未自动上传，**When** 我**手动选择录音文件上传**，**Then** 走同一条解析链路完成回填（BR-M4-01b，救济）。
  - **Given** 我是**其他服务商/非关联**催收员，**When** 我尝试操作该案件，**Then** 无入口/被拒（BR-M4-01；物业负责人/协调员对关联案件可打见 BR-M4-01a / US-M4-10）。
  - **Given** 本服务商分钟余额 ≤ CFG-MIN-WARN，**When** 我通话/回填，**Then** **通话与录音不受影响**，同时提示余额不足请充值（BR-M4-02a）。
  - **Given** 余额不足（≤ CFG-MIN-STOP），**When** 录音回填，**Then** 录音保存但**暂不自动解析**；充值后我可**手动触发解析**该录音（BR-M4-02a，M5）。

**US-M4-02 案件多联系方式管理与拨打**
- 作为**催收员**，我想管理案件的多个联系电话（标签/主号码/无效）并选择拨打，以便高效触达。
- 验收标准：
  - **Given** 案件含多个联系电话，**When** 我打开号码列表，**Then** 可选号码联系（跳起本机拨号器或自行拨打），主号码优先展示，每次通话独立记录（BR-M4-11、BR-M4-01b）。
  - **Given** 某号码，**When** 我给它打标签（本人/家属/代接人等）或设为主号码，**Then** 标签/主号码生效并随案保留（BR-M4-11、BR-M4-10）。
  - **Given** 某号码为空号/无效，**When** 我标注无效，**Then** 该号码在号码列表置灰、默认不作为联系/拨打对象（BR-M4-11）。

### Epic M4-B：标注跟进与承诺

**US-M4-03 写跟进记录 / 通话结果标记**
- 作为**催收员**，我想既能随时写自由跟进手记、又能对通话打结果标签，以便沉淀跟进且与状态变更区分清楚。
- 验收标准：
  - **Given** 我需要记录非状态性的跟进，**When** 我**写跟进记录**（跟进方式 + 内容，**可附图片/音频/文件**），**Then** 写入案件时间线·手记、**不改变案件状态**（BR-M4-03a）。
  - **Given** 一通通话已解析，**When** 我在**解析后 AI 复盘**里打通话结果标记，**Then** 按"是否有效跟进"决定是否重置 T_collector（BR-M4-03a、M5 BR-M5-04a）。
  - **Given** 一次**接通**通话后，**When** 我点选接通类标注并填小结，**Then** 生成有效跟进记录并重置 T_collector（BR-M4-03）。
  - **Given** 一次**未接通**外呼（未接/拒接/空号等），**When** 我标注，**Then** 仅留痕、**不重置 T_collector**（BR-M4-03）。
  - **Given** 标注枚举由平台配置，**When** 平台超管增改枚举及其"是否有效跟进"标记，**Then** 全平台标注选项随之更新（BR-M4-12、CFG-MARK-CODES）。
  - **Given** App 离线/弱网，**When** 我标注，**Then** 本地暂存、恢复网络后自动同步（OQ-M4-1 已定）。

**US-M4-04 登记承诺还款（含分期承诺）**
- 作为**催收员**，我想登记业主承诺还款（单笔或分期）并被提醒、被对照，以便盯紧兑现。
- 验收标准：
  - **Given** 业主承诺还款，**When** 我登记为**单笔**（金额+日期）或**分期**（每期金额+日期），**Then** 生成承诺型跟进记录并形成各期待跟踪项（BR-M4-13）。
  - **Given** 某期承诺临近到期，**When** 达预警提前量，**Then** 在我的**工作台显著位置**预警；到期再提醒（BR-M4-13）。
  - **Given** 业主实际回款，**When** 系统对照承诺各期，**Then** 逐期标记已兑现/部分兑现/违约，并沉淀为该案件履约记录（BR-M4-13）。
  - **Given** 某期违约，**When** 到期未回款，**Then** 标记违约并提醒我跟进。

### Epic M4-C：发缴费链接

**US-M4-05 发缴费链接给业主**
- 作为**催收员**，我想给业主发缴费链接（短信或转发微信），以便业主自助缴费。
- 验收标准：
  - **Given** 案件有减免后应收，**When** 我发送缴费链接，**Then** 链接金额=减免后应收（含分期取当期），带有效期（BR-M4-04/14）。
  - **Given** 我选短信送达，**When** 发送，**Then** 用平台统一签名+模板发出，按条扣减该物业短信条数；物业条数不足时提示（BR-M4-15/16）。
  - **Given** 我选复制/转发，**When** 复制链接到微信等渠道发给业主，**Then** 链接可用、**不走短信、不扣条数**（BR-M4-15）。
  - **Given** 链接已过期或被作废，**When** 业主访问，**Then** 提示失效，不可缴费；我可重发新链接（BR-M4-14）。
  - **Given** 业主打开链接，**When** 查看，**Then** 见应缴款信息 + 物业收款渠道（取自项目收款信息），**本期据此线下缴费**，回款由协调员标注（BR-M4-19、US-M4-08）。
  - **Given**（后期延伸）在线缴费上线，**When** 业主在线支付成功，**Then** 回款实时到账、系统自动结清、免协调员标注（BR-M4-19，M7）。

### Epic M4-D：转工单 / 建议法务

**US-M4-06 催收员转工单**
- 作为**催收员**，我想发起转工单要求，以便请物业协调员线下协调。
- 验收标准：
  - **Given** 我持有案件，**When** 我提交转工单（选**类型**+填诉求），**Then** 生成工单流向对应项目协调员，案件状态仍"进行中"且**仍留在我私海**（BR-M4-06/17）。
  - **Given** 协调员处理完工单，**When** 其回填结果，**Then** **回执给我**（结果+附件），我据此继续催收（BR-M4-17）。
  - **Given** 工单已转出，**When** 我查看案件，**Then** 看到工单状态，但案件不会因此自动结案。

**US-M4-07 催收员建议法务（轻量标注）**
- 作为**催收员**，我想对该走法务的案件打"建议法务"标注，以便提醒协调员关注，而不必交出案件。
- 验收标准：
  - **Given** 我持有某案件且认为应走法务，**When** 我打"建议法务"标注（可附理由），**Then** 生成跟进记录并**提醒对应协调员**，案件**仍在我私海、归属与状态不变**（BR-M4-05a）。
  - **Given** 我打了建议法务，**When** 我继续催收，**Then** 催收不中断；是否真正发起法务由协调员决定（BR-M4-18）。
  - **Given** 我不再持有该案件（释放/再分配），**When** 案件流转，**Then** "建议法务"标注随案保留，新持有人与协调员仍可见（BR-M4-10）。

### Epic M4-E：协调员处理

**US-M4-08 协调员标记线下回款**
- 作为**物业协调员**，我想标记业主线下/转账等非链接渠道的回款，以便回款数据完整。
- 验收标准：
  - **Given** 业主线下缴费，**When** 我登记金额/时间（**可选**上传凭证），**Then** 案件回款更新并参与 M8 结清判定与 M2 聚合，**持有催收员收到通知**（BR-M4-07）。
  - **Given** 登记的回款使减免后应收全部到账，**When** 保存，**Then** 触发 M8 自动结清结案（已结清）（BR-M4-07、BR-M8-01）。
  - **Given** 仅部分到账（含分期某期），**When** 保存，**Then** 记部分回款、案件仍"进行中"（BR-M4-07）。
  - **Given** 我误登记一笔回款，**When** 我冲正/撤销并填原因，**Then** 回款额回退、留痕；若曾因该笔结清则案件回退为"进行中"（BR-M4-07、M8 OQ-M8-5）。

**US-M4-09 协调员申请律师函/法律文书（按需付费，与催收并行）**
- 作为**物业协调员**，我想对关联范围内任意案件**按需申请律师函/法律文书**（催收函/律师函/要素化起诉状，**按次付费**），平台出具/打印后线下送达、签收拍照并存证，以便升级处置且不打断催收。
- 验收标准：
  - **Given** 我关联某案件（无论其在公海/私海/谁持有），**When** 我选择法务模板（催收函/律师函/要素化起诉状），**Then** 平台**用要素化起诉数据套模板自动生成 PDF**（可打印，BR-M2-15）；案件**不移出催收员私海、催收照常并行**（BR-M4-18）。
  - **Given** 文书已出具，**When** 我线下送达并**签收拍照上传**（App），**Then** 送达记录 + 照片关联案件并**生成送达存证**（M6 场景①）。
  - **Given** 我收到催收员的"建议法务"标注，**When** 我评估后发起或不发起法务，**Then** 决定权在我，且全程留痕（BR-M4-05a/18）。
  - **Given** 法务文书/送达/存证均产生，**When** 查看案件，**Then** 这些记录随案保留、关联人可见（BR-M4-10）。

**US-M4-09b 协调员处理工单回执**
- 作为**物业协调员**，我想处理转入工单并回执，以便闭环催收员的线下诉求。
- 验收标准：
  - **Given** 有转入工单，**When** 我处理并回填结果（含附件），**Then** 工单状态更新并**回执催收员**（BR-M4-17）。

**US-M4-10 物业侧给案件联系人打电话（不驱动池）**
- 作为**物业负责人/协调员**，我想给关联案件的联系人打 App 电话并录音，以便自行核实/沟通，且记录可被全程查看。
- 验收标准：
  - **Given** 我（物业负责人/协调员）关联某案件，**When** 我打 App 电话，**Then** 通话自动录音转写、**关联为案件跟进记录**，案件关联人可回放/看文本/取策略（BR-M4-01a）。
  - **Given** 我的物业有分钟余额，**When** 通话录音解析，**Then** 消耗本物业通话解析分钟数；余额不足只暂停自动解析、不影响打电话（BR-M4-02/02a）。
  - **Given** 我打了电话，**When** 系统处理，**Then** **不驱动公海/派单、不改变案件池归属、不重置持有催收员的 T_collector**（M3 BR-M3-08、总纲§6）。

### Epic M4-F：工作台与重复触达提示

**US-M4-11 催收员工作台与同业主提示**
- 作为**催收员**，我想在工作台聚合看到待办/提醒、并在案件详情看到同业主其他在催案件，以便高效跟进且不重复触达。
- 验收标准：
  - **Given** 我有承诺到期/释放预警/工单回执/新案件等待办，**When** 我打开工作台，**Then** 这些聚合展示并按紧急度排序（承诺/释放预警优先）（BR-M4-20）。
  - **Given** 同一业主在我本服务商可见范围内另有在催案件，**When** 我打开案件详情，**Then** 看到"同业主其他在催案件"提示（BR-M4-21）。

**US-M4-12 通话记录查询**

  - **Given** 我打开「通话记录」，**When** 按业主/房号/电话搜索或按项目/批次/时间过滤，**Then** 命中我可见范围内的通话列表（BR-M4-22）。
  - **Given** 某条通话记录，**When** 我点击，**Then** 可打开该通 **AI 复盘**（对话记录+质检+下一步建议，M5 BR-M5-04a）或进入案件详情。
  - **Given** 案件已结案，**When** 我（服务商/催收员）查询，**Then** 按 M8 BR-M8-09 收敛（不展示明细）。
