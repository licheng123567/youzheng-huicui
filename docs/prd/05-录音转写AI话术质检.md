# 有证慧催 PRD · M5 录音转写 + AI 话术 + 质检（风控）

> 依赖：[00-总纲](./00-总纲.md) §1 护城河、§2 计费、[M2](./02-项目批次案件主数据.md)（作战手册）、[M4](./04-催收作业.md)（通话录音是输入）。
> 主要角色：**全员**（平台 / 物业 / 服务商）。
> 目标：定义通话录音的**语音解析（STT）**与其上的四项 AI 能力——**①逐通话策略建议 ②作战手册 AI 初稿 ③有效话术自动提炼 ④风险检测（质检/风控）**，以及按分钟的计费/补解析。
> **契约对齐**：本模块与 API 契约 v1.0.0 对齐。

---

## 1. 模块范围

- 通话录音 → **语音解析（STT 转文字）**，按通话解析分钟数计费；余额不足暂停自动解析、可手动补解析。
- **四项 AI 能力（基于转写文本，平台增值）**：
  1. **逐通话策略建议**：通话前依画像、通话后据转写，给下一步催收策略。
  2. **作战手册 AI 初稿**：聚合项目历史通话，自动生成/迭代作战手册初稿，**物业负责人审核采纳**。
  3. **有效话术自动提炼**：从录音/转写自动提炼高转化话术，沉淀**话术库**，反哺 1 与 2。
  4. **风险检测（质检/风控）**：自动识别辱骂/脏话/威胁/违规承诺等风险，分级告警，**物业/服务商/平台三方按隔离可见**，并可生成处置待办。

**不在本模块**：通话发起/录音本身（M4）、计费充值与对账明细（M9）、存证（M6）。

## 2. 关键概念

| 概念 | 说明 |
|---|---|
| 语音解析（STT） | 录音上传平台转文字；按通话解析分钟数计费（CFG-STT-PRICE） |
| 自动解析 / 手动补解析 | 有余额时通话后自动解析；余额不足暂停，充值后可手动触发补解析 |
| 用户画像 | 业主/案件维度画像（欠费历史、履约、沟通态度等），驱动策略建议 |
| 策略建议 | 逐通话的下一步催收策略（通话前 + 解析后 AI 复盘；不做通话中实时提示） |
| 作战手册 AI 初稿 | AI 据项目历史通话生成的作战手册草稿，需物业负责人审核采纳 |
| 话术库 | 自动提炼的高转化话术沉淀，**平台级、平台所有（护城河资产），不对物业/服务商开放**；对外仅输出 AI 策略与话术建议 |
| 风险检测 | 对转写/录音识别违规风险（辱骂/脏话/威胁/违规承诺/泄露等），分级告警 |

## 3. 业务规则

### 3.1 语音解析与计费
- **BR-M5-01（计费对象）**：计费/扣减发生在**语音解析（STT）**环节，按**通话解析分钟数**（CFG-STT-PRICE），**计入发起通话方所属组织**。**打电话+录音不消耗分钟、不受余额限制**（重申 M4 BR-M4-02）。
- **BR-M5-02（自动 vs 手动解析）**：有余额 → 通话后**自动解析**；余额 ≤ CFG-MIN-STOP → **暂停自动解析**（录音照常保存，录音记录状态 = `CallRecStatus.QUOTA_BLOCKED`），**充值后可对未解析录音手动补解析（支持批量）**；余额 ≤ CFG-MIN-WARN 预警（M4 BR-M4-02a）。
  - **批量补解析按录音时间顺序进行，解析到余额扣完为止**；余额不足以覆盖剩余的录音留在"待解析"，提示继续充值。
- **BR-M5-03（转写归属）**：转写文本归属**案件**、与录音关联，案件关联人（有权者）可查看；**结案后对服务商/催收员按 M8 BR-M8-09 收敛为只看统计**。
- **BR-M5-11（说话人分离）**：转写做**说话人分离**（区分催收员/业主等），以**对话形式（类似微信聊天气泡）按时间线呈现**，便于阅读、风险定位与质检；可点对话片段定位到录音对应时间点（TBD）。
- **BR-M5-10（AI 分析计费口径）**：四项 AI 能力为**平台增值能力，本期不单独计费**（含在平台能力内）；仅 STT 解析按分钟计费。是否对 AI 分析单独计费 = OQ-M5-6。

### 3.2 四项 AI 能力
- **BR-M5-04（逐通话策略建议）**：策略建议**综合多源**——**案件业主综合画像**（欠费金额/账龄、历史沟通态度、履约/承诺兑现率、联系可达性、是否争议等）+ **本案件历史全部通话/跟进** + **话术库（BR-M5-06）** + **项目作战手册（M2）**。**通话前**给话术建议与策略（载体=作战手册，**AI 动态策略在上、物业静态资料在下**）；**通话后（录音解析完成）**据本通话+历史给下一步策略（如承诺跟踪、建议法务、换号触达等）。策略为辅助、不强制；对**全员**按权限开放。**不做通话中实时提示**（系统不主动拨号、录音为通话后回填，无通话中链路；OQ-M5-1）。
- **BR-M5-04a（解析后 AI 复盘）**：每通录音解析完成后提供**"AI 复盘"**视图（案件详情**右侧抽屉、宽度可调**），含——**①对话记录**（说话人分离、微信式气泡：催收/协调员在右·绿色，业主在左·白色，BR-M5-11）；**②本通质检风险点**（L1/L2，可"通知督导 / 转质检"，BR-M5-07）；**③下一步建议**（据本通+画像+话术库+作战手册生成，BR-M5-04），建议卡**可"采纳"并直接联动**对应动作弹窗（登记承诺 / 建工单 / 发催费单 / 写跟进记录 BR-M4-03a）；并在复盘内完成**通话结果标记**（BR-M4-03a）。复盘内容**按案件差异化**。复盘入口由 M4"获取最新通话录音"回填后直达（BR-M4-01b）。
- **BR-M5-05（作战手册 AI 初稿 + 人审 + 增量更新）**：作战手册初稿由 AI 生成，**输入综合**——①物业**录入的项目相关情况**（小区特点、欠费背景、历史难点、特殊政策等）；②**平台话术库**（BR-M5-06）；③**项目历史通话**；经 AI 能力**生成优化后的作战手册初稿与话术**。初稿须经**物业负责人审核采纳**方为正式作战手册（M2）；支持版本迭代与回溯，AI 不直接对外发布。
  - **增量重生成**：随通话积累，系统**提示"作战手册可更新"**，物业可**一键生成新版**并与现行版**对比**（差异高亮）后选择采纳，形成版本演进。
  - **BR-M5-05a（采纳人 = 物业负责人或协调员）**：作战手册新版的对比/采纳/忽略，**物业负责人与物业协调员均可决定**（话术更新是物业内部事务）；服务商/催收员只读。
  - **BR-M5-05b（分级采纳闸 + 提醒）**：版本演进采用**分级采纳闸**（平台 AI 配置中心 BR-M5-13 可配）——①默认 **强制人工采纳**（物业采纳才发布）；②物业可按项目**自愿开启"低风险自动采纳"**（仅措辞/排序微调、保留版本、可一键回滚）；③**敏感改动（减免/法律/承诺口径）始终人工**。"作战手册可更新"**三处提醒**物业：工作台今日必办、消息中心推送、项目档案/作战手册区红点。**话术库（内部）始终系统自动进化**，不受采纳闸约束（仅输出建议、不直接面客）。
- **BR-M5-06（有效话术自动提炼 + 话术库）**：系统**自动**从录音/转写提炼"有效话术"（与回款/承诺转化相关），沉淀为**话术库**（按项目/场景分类），**反哺策略建议（BR-M5-04）与作战手册初稿（BR-M5-05）**。
  - **话术库是平台级、平台所有的护城河资产**：平台可跨项目/跨物业全局聚合沉淀；**话术库本身不对物业/服务商开放**，对外仅输出"AI 策略与话术建议"。作战手册初稿（经物业采纳后归物业项目）属衍生输出，非话术库本体。
  - **BR-M5-06a（话术库两类来源 + 后台维护 + 效果评估）**：话术库条目来源有二——①**AI 飞轮自动提炼**；②**专家/平台手工录入**（平台后台「平台话术库」录入场景/意图/适用 cohort/话术内容）。二者统一沉淀、统一按场景供 AI 策略建议（BR-M5-04）与作战手册初稿（BR-M5-05）调用，专家话术经此间接惠及一线（催收员/协调员），**不破坏"原始库不外泄"**。
    - **可见可管**：「平台话术库」**仅平台（超管/质检员）可见可管**，物业/服务商/催收员均不见原始库。
    - **效果评估可查询**：每条话术记录 **使用次数 / 承诺转化率 / 回款转化率 / Wilson 下界 / 状态（生效·候选·淘汰）**，可按 **场景 / 来源 / 状态** 筛选查询；转化数据由真实通话结果回流统计（飞轮 BR-M5-12），专家新录条目初期"待积累"、随使用补齐。
    - **生命周期**：候选 →（达阈值或人工）生效 → 低效**淘汰**（停止供 AI 调用），全程留痕。
- **BR-M5-12（话术有效性评判与优化闭环 = 飞轮）**：
  - **有效性以"话术 ↔ 结果"关联衡量**，信号分层——**短期**：接通、拿到承诺、业主态度/情绪变化（转写+情绪识别）；**中期**：承诺兑现、是否回款、回款时长。
  - **形成**：从海量转写按**场景**（首次接触/失联/拒缴争议/承诺协商/分期等）抽取话术片段 → 统计各话术关联的**承诺转化率 / 回款转化率** → 达阈值沉淀入平台话术库，低效话术标记/淘汰。
  - **优化闭环（飞轮）**：通话 → 转写 → 结果标注（承诺/回款）→ 话术效果统计 → 更新话术库 → 反哺策略建议（BR-M5-04）与作战手册初稿（BR-M5-05）→ 下一次通话……相似画像下可做话术 A/B 对比。
  - **更新节奏**：**策略建议按每通（通话前 + 解析后复盘）**生成（用当前话术库快照；非通话中实时提示，OQ-M5-1）；**话术库为准实时/批量增量更新**（积累到阈值 + 定期触发，非每通实时重算）；作战手册为物业触发的增量重生成。
  - **BR-M5-12a（话术级自我迭代 + 变体晋升闸）**：飞轮不止统计/淘汰，还**主动改写**——AI 通话后分析"话术↔结果"，对每条话术**自动生成优化变体**，相似 cohort 下与现行做 **A/B 实测**，按效果决定是否替换。**是否更新由晋升闸控制（按来源分级）**：①**AI 提炼话术**=数据达标**自动晋升**；②**专家/平台录入话术**=AI 只产**变体候选**、默认**人工复核**才替换；晋升**保留旧版可回滚**。后台（平台 AI 配置中心 BR-M5-13 ③）可配 **自动优化变体开关 / A/B 实测开关 / 晋升策略**（专家复核·AI自动 ↔ 全自动可回滚 ↔ 全人工）。因话术库是**内部资产、只输出建议、可回滚**，故可比作战手册（面客、物业担责）更自动地自我迭代。
  - 具体评分算法、转化阈值、场景分类初始集、模型选型 = OQ-M5-8 / OQ-M5-9。
- **BR-M5-13（平台 AI 能力配置中心 = 后台调优）**：平台超管在「参数配置」维护 AI 能力底座与策略，分三层（SE 仅在显式授权下可见/可写，01§8）：
  - **① 模型与密钥**：LLM=**DeepSeek 直连**（API Key 密文存储、模型 deepseek-chat / deepseek-reasoner、温度 / 最大Tokens / 超时）；ASR=**阿里百炼 paraformer-8k-v2**（API Key、8k 电话语音、**热词表**、说话人分离）。Key 仅平台超管可见可改、密文存储。
  - **② 通话前 / 通话后策略（提示词与调优）**：通话前 = 作战手册话术指引提示词（BR-M5-04）；通话后 = AI 复盘提示词（摘要 + 质检风险 + 下一步建议，BR-M5-04a），可配 **质检风险规则集（CFG-RISK-TYPES）/ 建议条数 / 温度**；**通话中实时提示恒为关闭**（OQ-M5-1）。
  - **③ 作战手册飞轮后台**：配置 **自动迭代开关 / 回流来源（质检风险 · 高回款话术 · 高频异议）/ 迭代触发节奏（每周 · 每月 · 每 N 通 · 手动）/ 版本保留数 / 分级采纳闸**（强制人工采纳 ↔ 低风险物业自愿自动·可回滚，敏感始终人工，BR-M5-05b）——AI 只产草稿，发布权恒在物业（负责人或协调员，BR-M5-05a）。平台只调飞轮参数与提示词模板，**不替物业决定话术内容**。
- **BR-M5-07（风险检测 = 质检/风控）**：**所有已解析的通话录音均做风险检测**（含催收员通话与物业负责人/协调员通话），自动识别**风险**（**风险类型可配 CFG-RISK-TYPES，枚举待定**），按**高/中/低分级**；**高风险准实时告警**。风险记录归属案件/操作人，并**精确关联到对应录音及对话片段/时间点（BR-M5-11）**，便于回放取证。
  - **三方按隔离可见**：物业看**本项目**风险、服务商看**本服务商**风险、平台看**全局**。
  - **高风险可生成风险处置待办**（要求服务商整改/培训），形成闭环；处置留痕。
  - **BR-M5-07a（处置归属 = 谁的员工谁处理）**：风险的**实质处置**（整改 / 培训 / 警告 / 停权）**由违规人所属组织的负责人执行**——**催收员违规 → 服务商负责人**处置；**物业协调员违规 → 物业负责人**处置（违规管理是各自组织内部事务，平台只考核到组织，M10 BR-M10-09）。**对"非本组织员工"的风险仅"只读查看 + 上报"**（如物业看到催收员风险→上报服务商/平台，不直接处罚）。平台质检可介入复核。
  - **BR-M5-07b（处置任务跟踪 = 平台监管视图）**：「**风险处置任务跟踪（整改/培训闭环）**」是**跨组织的监管闭环视图，仅平台（超管/质检员）可见**；**服务商与物业两侧均不查看**此跟踪表（各自只在质检看板处置/上报本侧风险）。平台据此跟踪各组织整改完成状态。
  - **BR-M5-07c（平台质检复核）**：平台（超管/质检员）对风险做**复核判定**——结论分 **确认风险（真实）/ 判定误报（撤销该风险）/ 升级平台处置**，复核留痕。**平台只复核、不替代组织处置**——实质处置（整改/培训/停权）仍归违规人所属组织负责人（BR-M5-07a）。组织侧质检看板对平台显示「复核」入口，而非处置动作。

### 3.3 通用
- **BR-M5-08（解析失败处理）**：解析失败（音质差/格式问题）须可见失败原因、**可重试**；失败**不扣分钟**（口径 TBD，OQ-M5-3）；录音仍保留。
- **BR-M5-09（敏感信息与合规）**：转写/录音含业主敏感信息，受 M1 隔离与 M8 结案收敛约束；风险检测/话术提炼/手册生成的**训练与样本使用须脱敏、在合规范围内**（TBD）。

## 4. Epic 与用户故事

### Epic M5-A：语音解析（STT）

**US-M5-01 通话录音自动解析**
- 作为**催收员/物业用户**，我想通话后录音自动转文字，以便复看与驱动 AI 能力。
- 验收标准：
  - **Given** 本组织有分钟余额，**When** 通话结束录音落库，**Then** 自动解析为文字，按时长扣减本组织分钟（BR-M5-01/02）。
  - **Given** 转写完成，**When** 我查看案件，**Then** 见转写文本（**说话人分离、类似微信对话气泡按时间线**）并可对照录音回放（BR-M5-03、BR-M5-11）。

**US-M5-02 余额不足与手动补解析**
- 作为**组织用户**，我想余额不足时仍能打电话、充值后补解析，以便不中断催收。
- 验收标准：
  - **Given** 余额 ≤ CFG-MIN-STOP，**When** 通话结束，**Then** 录音保存、列入"待解析"、暂不自动解析（BR-M5-02）。
  - **Given** 我充值后，**When** 对待解析录音手动触发解析，**Then** 解析完成并扣减分钟（BR-M5-02）。
  - **Given** 有多条待解析录音，**When** 我批量补解析，**Then** **按时间顺序解析到余额扣完为止**，余额不足覆盖的留在"待解析"并提示继续充值（BR-M5-02）。

### Epic M5-B：策略建议与话术飞轮

**US-M5-03 逐通话策略建议**
- 作为**催收员**，我想通话前后都拿到策略建议，以便提高成功率。
- 验收标准：
  - **Given** 我准备拨打，**When** 打开话术辅助，**Then** 系统综合**案件业主综合画像**+本案件历史通话+话术库+作战手册给话术建议与策略（BR-M5-04）。
  - **Given** 一通通话已解析，**When** 我打开**AI 复盘**，**Then** 见对话记录（微信式气泡）+ 本通质检风险点 + 下一步建议，建议卡可**采纳并联动**对应动作弹窗，并可在复盘内打通话结果标记（BR-M5-04a、BR-M4-03a）。
  - **Given** 通话产生承诺/回款等结果，**When** 结果回流，**Then** 计入话术有效性统计、反哺话术库与后续建议（飞轮，BR-M5-12）。

**US-M5-04 作战手册 AI 初稿 + 物业审核采纳**
- 作为**物业负责人**，我想让 AI 据项目通话生成作战手册初稿、我审核后采纳，以便低成本沉淀打法。
- 验收标准：
  - **Given** 项目已有一定量通话转写，**When** 我**录入项目相关情况**并请求生成作战手册初稿，**Then** 系统综合"我录入的情况 + 平台话术库 + 历史通话 + AI"生成优化后的初稿与话术（BR-M5-05）。
  - **Given** 我审阅初稿，**When** 我编辑并采纳，**Then** 成为该项目正式作战手册（M2），并记录版本；**未采纳前 AI 初稿不对服务商正式发布**（BR-M5-05）。
  - **Given** 通话持续积累，**When** 系统提示"作战手册可更新"（工作台今日必办 + 消息中心 + 项目档案红点三处提醒），**Then** 我可一键生成新版并与现行版对比（差异高亮），选择采纳形成新版本（BR-M5-05/05b）。
  - **Given** 我是**物业负责人或协调员**，**When** 对作战手册新版对比/采纳/忽略，**Then** 均可决定（话术更新是物业内部事务）；服务商/催收员只读（BR-M5-05a）。
  - **Given** 采纳闸分级，**When** 版本演进，**Then** 默认**强制人工采纳**；物业可按项目自愿开**低风险自动采纳**（仅措辞/排序微调、保留版本可回滚）；**敏感改动（减免/法律/承诺口径）始终人工**（BR-M5-05b）。

**US-M5-05 话术库提炼 + 专家录入 + 自我迭代（平台）**
- 作为**平台（超管/质检员）**，我想让话术库既能 AI 自动提炼、也能专家录入，并能查效果、自我迭代，以便反哺建议与手册且不外泄库本体。
- 验收标准：
  - **Given** 积累大量转写，**When** 系统分析话术与转化相关性，**Then** 提炼有效话术入**平台话术库**（平台所有，按项目/场景分类）（BR-M5-06）。
  - **Given** 我在「平台话术库」后台，**When** 我录入**专家话术**（场景/意图/适用 cohort/内容），**Then** 与 AI 提炼话术统一沉淀、统一供 AI 调用，专家话术经"AI 建议 + 作战手册"间接惠及一线（BR-M5-06a）。
  - **Given** 我查看话术库，**When** 浏览，**Then** 见每条**使用次数 / 承诺转化率 / 回款转化率 / 状态（生效·候选·淘汰）**，可按场景/来源/状态筛选；低效可淘汰（BR-M5-06a）。
  - **Given** AI 通话后分析话术↔结果，**When** 自动产出优化变体并 A/B 实测，**Then** **AI 提炼话术达标自动晋升、专家话术 AI 只产候选需人工复核**，晋升保留旧版可回滚（BR-M5-12a）。
  - **Given** 我是物业/服务商，**When** 我使用 AI 能力，**Then** 仅得到 AI 策略与话术建议，**不可见/不可导出话术库本体**（BR-M5-06）。

**US-M5-07 平台 AI 能力配置中心**
- 作为**平台超管**，我想统一配置 AI 能力底座与策略，以便 AI 越用越准且物业可控。
- 验收标准：
  - **Given** 我在参数配置「AI 能力配置中心」，**When** 维护，**Then** 可配 **LLM（DeepSeek 直连：Key/模型/温度/Tokens）+ ASR（百炼 paraformer-8k-v2：Key/热词/说话人分离）**，Key 密文存储（BR-M5-13）。
  - **Given** 配置通话前/后策略，**When** 维护提示词，**Then** 可配 **通话前作战手册指引 + 通话后 AI 复盘提示词 + 质检风险规则集 + 建议条数/温度**；**通话中实时提示恒为关闭**（BR-M5-13、OQ-M5-1）。
  - **Given** 配置作战手册飞轮，**When** 维护，**Then** 可配 **自动迭代开关 / 回流来源 / 触发节奏 / 版本保留数 / 分级采纳闸**；AI 只产草稿、发布权恒在物业（BR-M5-13、BR-M5-05a/b）。
  - **Given** 我是平台员工(SE)，**When** 访问 AI 配置中心，**Then** 默认不授权、需超管显式授予，未授权为只读（01 §8、BR-M1-04b）。

### Epic M5-C：风险检测（质检/风控）

**US-M5-06 风险检测与三方告警处置**
- 作为**平台/物业/服务商**，我想自动识别催收通话风险并按角色看到、可处置，以便管控合规（护城河）。
- 验收标准：
  - **Given** **任一角色**（催收员/物业）的通话已转写，**When** 风险检测运行，**Then** **所有录音均检测**，识别风险类型（可配）并分高/中/低级，关联案件/操作人（BR-M5-07）。
  - **Given** 命中风险，**When** 我查看，**Then** 风险**精确关联到对应录音与对话片段/时间点**，可一键回放定位（BR-M5-07、BR-M5-11）。
  - **Given** 命中**高风险**（如辱骂/威胁），**When** 检出，**Then** **准实时告警**，并按隔离推送：物业见本项目、服务商见本服务商、平台见全局（BR-M5-07）。
  - **Given** 高风险产生，**When** 平台/物业发起处置，**Then** 生成风险处置待办（要求服务商整改/培训），处置留痕闭环（BR-M5-07）。
  - **Given** 我是服务商，**When** 查看风险，**Then** 仅见本服务商风险，不可见他商（隔离，BR-M5-07）。
