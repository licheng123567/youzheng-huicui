# 有证慧催 PRD · M7 业主缴费 H5

> 依赖：[00-总纲](./00-总纲.md) §3 三端、[M2](./02-项目批次案件主数据.md)（应收/收款信息/减免）、[M4](./04-催收作业.md)（缴费链接 BR-M4-04/14/15/19）。
> **契约对齐**：本模块与 API 契约 v1.0.0 对齐。
> 形态：**业主无登录 H5**（扫码 / 缴费链接进入）。
> 目标：定义业主侧账单查看与缴费——**本期：展示应缴 + 物业收款渠道、线下缴、协调员标注；在线缴费为后期延伸**。

---

## 1. 模块范围

- **本期最小实现**：缴费链接 = **短信文字 + 一个 H5 静态展示页（只读）**。
- H5 静态页**查看账单/应缴明细**（减免后应收、含分期当期）+ **展示物业收款渠道**（取自项目收款信息）。
- 业主据此**线下缴费**，协调员标注回款；**业主不在 H5 上传凭证**。
- 链接有效期/失效提示（BR-M4-14）。
- **后期优化**：扫码/在线缴费（在线支付、实时到账、自动结清、免标注）。

**不在本模块**：缴费链接发送（M4）、回款标注与结清（M4/M8）、结算（M9）。

---

## 2. 关键概念

| 概念 | 说明 |
|---|---|
| 业主 H5 | 无登录网页，凭缴费链接/扫码进入，仅展示该案件账单与缴费入口 |
| 应缴明细 | 减免后应收（含分期则按期展示当期/全部期次） |
| 收款渠道 | 物业↔业主收款账户/通道，取自项目"收款信息"（M2） |
| 在线缴费（后期） | 业主在线支付、实时到账、自动结清——本期不实现 |

---

## 3. 业务规则

- **BR-M7-01（无登录静态展示页）**：业主凭**短信中的缴费链接**进入 **H5 静态展示页（只读）**，无需登录；链接仅暴露**该案件**账单与收款渠道，不可越权访问他案（凭 token，TBD）。**本期不做扫码、不做在线支付、业主不上传凭证**（均后期）。
- **BR-M7-02（账单展示）**：H5 展示**减免后应收**（含分期则展示各期与当期应缴）、欠费周期、物业费说明等（金额口径同 BR-M2-05）。
- **BR-M7-03（本期缴费 = 静态展示 + 线下）**：本期 H5 静态页**展示应缴 + 物业收款渠道**（取自项目收款信息）；**短信文字直接带关键信息**——**应缴金额 + 收款方式摘要 + H5 链接**，业主不点链接也能看到核心；业主据此**线下缴费**；回款由**协调员标注**（M4 US-M4-08）→ 触发结清判定（M8）。**业主不在 H5 上传凭证**，由协调员线下核实标注。（短信受物业短信通道/条数约束，M4 BR-M4-16）
- **BR-M7-04（链接有效期）**：过期/作废链接访问时**提示失效**、不可缴费（BR-M4-14）。
- **BR-M7-05（在线缴费，后期）**：在线缴费上线后，业主在线支付成功 → **回款实时到账、系统自动结清、免协调员标注**（BR-M4-19）。在线缴费的支付通道/对账归 M9。
- **BR-M7-06（分期展示）**：案件为分期时，H5 展示分期计划与**当期应缴**；此处分期指**政策分期**（来自项目减免规则，M2 BR-M2-16），非催收员的承诺分期。分期审批：规则内自动；超自决档走线下内部流程（系统仅留痕）或物业负责人系统内核准，**无协调员特批/系统审批流**（M2 BR-M2-18a，OQ-2 已定）。
- **BR-M7-07（隐私最小化）**：H5 只展示缴费必要信息（金额/周期/收款渠道），**不暴露催收过程/其他案件/服务商信息**。

## 4. Epic 与用户故事

### Epic M7-A：账单查看

**US-M7-01 业主查看账单明细**
- 作为**业主**，我想打开链接看到欠费账单明细，以便了解应缴。
- 验收标准：
  - **Given** 我收到有效缴费链接，**When** 我打开 H5，**Then** 看到减免后应收、欠费周期、物业费说明（无需登录）（BR-M7-01/02）。
  - **Given** 案件为分期，**When** 我查看，**Then** 看到分期计划与当期应缴（BR-M7-06）。
  - **Given** 我打开 H5，**When** 查看缴费方式，**Then** 同时看到**物业收款渠道/收款信息**（取自项目收款信息），据此线下缴费（BR-M7-02/03）。
  - **Given** 链接已过期/作废，**When** 我打开，**Then** 提示失效、不可缴费（BR-M7-04）。

### Epic M7-B：本期缴费（展示 + 线下）

**US-M7-02 线下缴费 → 协调员标注闭环**
- 作为**业主**，我想据展示的收款渠道线下缴费、由协调员确认入账，以便完成缴费闭环。
- 验收标准：
  - **Given** 我已线下缴费，**When** 协调员核实并标注回款（M4），**Then** 案件回款更新、触发结清判定（M8）。
  - **Given** 我打开 H5，**When** 浏览，**Then** 不可见催收过程/其他案件/服务商信息（BR-M7-07）。

### Epic M7-C：在线缴费（后期延伸）

**US-M7-03 业主在线缴费（后期占位，本轮不实现）**
- 作为**业主**，我想直接在线支付，以便即时完成缴费。
- 验收标准：
  - **Given** 在线缴费已上线，**When** 我在线支付成功，**Then** 回款实时到账、系统自动结清、无需协调员标注（BR-M7-05）。
  - **Given** 在线支付，**When** 涉及对账，**Then** 入 M9 在线缴费对账流程。
