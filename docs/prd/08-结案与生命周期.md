# 有证慧催 PRD · M8 结案与生命周期

> 依赖：[00-总纲](./00-总纲.md) §6 生命周期、[M3 流转](./03-案件流转与公海.md)、[M4 催收作业](./04-催收作业.md)。
> **契约对齐**：本模块与 API 契约 v1.0.0 对齐。
> 目标：定义案件的终态——结清结案、撤案/坏账结案、长期无回款兜底、结案后可见性收敛（服务商/催收员只看统计），以及谁可结案的权责边界。

---

## 1. 模块范围

- 结清自动结案（应收全部到账）。
- 撤案/坏账结案（非回款关闭，物业协调员/负责人决定 + 平台代操作，不设审核流）。
- 谁不能结案（催收员/服务商）。
- 长期无回款兜底（最长催收周期提醒，不自动结案）。
- 结案后可见性收敛（服务商/催收员只看统计、不看明细）。

**不在本模块**：结算/对账（M9）、回款登记动作本身（M4/M7）。

---

## 2. 案件终态与生命周期

```
进行中
  ├─ 应收(减免后/分期)全部到账 ──系统自动──► [已结清]
  ├─ 协调员/负责人决定(须选原因) ───────────► [已撤案 / 坏账]
  │     (业主搬离/查无此人/无催收价值/物业撤案等; 协调员/负责人操作, 平台超管可代操作)
  └─ 入池超"最长催收周期"仍无任何回款 ──► 系统提醒物业负责人处置(不自动结案)
                                            └► 物业负责人选择: 结案 / 继续 / 发起法务

结案后 ──► 服务商/催收员只见统计、不见明细；案件明细仅平台+物业可见
```

---

## 3. 业务规则

### 3.1 结清结案
- **BR-M8-01**：当案件**减免后应收**（含**政策分期**则全部期次，M2 BR-M2-16）**全部到账** → 系统**自动结案为"已结清"**，无需人工。（承诺分期不参与结清判定）
- **BR-M8-02**：回款来源包括业主 H5 缴费（M7）与协调员登记的线下回款（M4 BR-M4-07），二者合并计入应收到账。
- **BR-M8-11（分期承诺与释放/兜底的关系）**：
  - 案件存在**进行中的有效分期承诺**（未违约）时，**暂缓 T_collector 自动释放**（M3 BR-M3-08），改由**承诺到期预警**（M4 BR-M4-13）驱动催收员逐期跟进；**某期违约 → 恢复 T_collector 计时**。
  - 分期**已有部分回款**的案件**不触发"无任何回款"最长周期兜底**（BR-M8-08）。
  - **分期跨催收员时的佣金归属**（中途释放/换人后最终回款归谁）= **M9 OQ**（待结算细化）。

### 3.2 撤案 / 坏账结案
- **BR-M8-03**：非回款关闭（业主搬离/查无此人/无催收价值/物业撤案等）为**撤案/坏账结案**，**物业协调员或物业负责人均可直接操作**，**须选择原因**（操作留痕到人）。平台超管可代操作（BR-M8-04）。
- **BR-M8-04**：**平台超管可代操作**撤案/坏账结案（**不设审核流**；代操作须留痕、被代物业事后可见，M1 BR-M1-15/US-M1-07）。
- **BR-M8-05**：撤案/坏账原因为受控枚举（**枚举集待定 TBD**），**由平台超管统一配置**（CFG-CLOSE-REASONS）；选"其他"时**须填备注**。
- **BR-M8-12（撤案监控防滥用）**：撤案/坏账行为全部留记录（操作人/原因/时间）；**物业负责人可见本物业、平台可见全平台的撤案频次与原因分布统计**（类比释放记录 M3 BR-M3-27），用于发现异常核销、防权限滥用。

### 3.3 谁不能结案
- **BR-M8-06**：**催收员/服务商不能结案**；只能释放回公海、标注跟进、打"建议法务"标注（法务由协调员主导，M4 BR-M4-18）。
- **BR-M8-07**：**释放回公海 ≠ 结案**——案件仍"进行中"（重申 M3 BR-M3-10）。

### 3.4 长期无回款兜底
- **BR-M8-08**：案件入池超过**最长催收周期（CFG-MAXCYCLE，超管可配，按项目配置）**仍未结清 → 系统**提醒物业负责人处置**（结案/继续/发起法务），**不自动结案**。提醒**分两类标注**：①**无任何回款**；②**有过回款但仍未结清**（如部分缴费后僵滞）。两类都提醒，仅分类不同以便判断。

### 3.5 结案后可见性收敛
- **BR-M8-09（结案即只看统计）**：案件一**结案（结清/撤案/坏账）**，**服务商与催收员立即不可见任何案件明细**（业主信息/联系方式/跟进/录音/转写/法务文书等），**仅可见统计聚合**（件数/回款额/回款率/佣金/绩效等）；**案件明细结案后仅平台（质检/审计）与物业（数据所有方）可见**。**无 30 天过渡期**（取代原隐私窗口/脱敏机制）。
- **BR-M8-09a（本人业绩统计口径例外）**：在 BR-M8-09 "只看统计" 之上，催收员可在**「我的业绩」**查看**本人**已结案案件的**回款明细**——**房号 + 脱敏姓名 + 回款额 + 结清日期 + 个人提成**，用于核对个人回款户数与提成。该明细**仍不含联系方式/跟进/录音，且不可再进入案件详情**（无法据此再次触达业主），不破坏脱敏意图。**仅限"本人 + 统计核对口径"**，不得扩展为跨人/业务明细访问。
- **BR-M8-10**：结案为终态——案件**不再参与公海/抢单/派单**；案件明细供**平台/物业**用于对账/报表/审计与存证追溯（M9/M10/M6）。

## 4. Epic 与用户故事

### Epic M8-A：结清自动结案

**US-M8-01 应收全部到账自动结清**
- 作为**平台（系统）**，我想在案件减免后应收全部到账时自动结案为"已结清"，以便无需人工干预。
- 验收标准：
  - **Given** 案件减免后应收未结清，**When** 一笔回款使累计到账 = 减免后应收，**Then** 系统自动将案件置"已结清"并记录结案时间/触发来源（BR-M8-01/02）。（注：**本期回款唯一入口=协调员线下标注**（US-M4-08）；H5 在线到账自动结清属**后期延伸**，BR-M7-03/M4-19。）
  - **Given** 案件为分期，**When** 仅部分期次到账，**Then** 不结清，案件仍"进行中"。
  - **Given** 分期最后一期到账使全部期次结清，**When** 到账，**Then** 自动结清。
  - **Given** 案件自动结清，**When** 结案完成，**Then** 对服务商/催收员收敛为只看统计（BR-M8-09）。

### Epic M8-B：撤案 / 坏账结案

**US-M8-02 物业协调员/负责人撤案/坏账结案**
- 作为**物业协调员或物业负责人**，我想对无回款价值的案件做撤案/坏账结案并选择原因，以便清理案件。
- 验收标准：
  - **Given** 案件"进行中"且我是物业协调员或负责人，**When** 我选择撤案/坏账并选定原因，**Then** 案件结案为对应终态并记录原因/操作人/时间（BR-M8-03/05）。
  - **Given** 我未选择原因，**When** 我提交，**Then** 系统阻止并要求选原因。
  - **Given** 案件结案，**When** 完成，**Then** 对服务商/催收员收敛为只看统计（BR-M8-09）。
  - **Given** 撤案/坏账已发生，**When** 物业负责人/平台查看统计，**Then** 可见撤案频次与原因分布，便于发现异常核销（BR-M8-12）。

**US-M8-03 平台超管代操作结案**
- 作为**平台超管**，我想在必要时代物业执行撤案/坏账结案，以便兜底支持（不设审核流）。
- 验收标准：
  - **Given** 物业未能自行处理，**When** 我代操作撤案/坏账（须选原因），**Then** 结案成功且记录"平台代操作 + 代谁 + 操作人 + 时间"，**被代物业事后可见**（BR-M8-04、BR-M1-15）。

**US-M8-04 催收员/服务商不可结案**
- 作为**平台**，我想确保催收员/服务商无法结案，以便权责清晰。
- 验收标准：
  - **Given** 我是催收员/服务商，**When** 我查看案件操作，**Then** 无"结案"入口，仅有释放/标注/建议法务（BR-M8-06）。
  - **Given** 催收员释放案件回公海，**When** 释放完成，**Then** 案件仍"进行中"，未结案（BR-M8-07）。

### Epic M8-C：长期无回款兜底

**US-M8-05 最长催收周期提醒**
- 作为**平台（系统）**，我想在案件超过最长催收周期仍未结清时提醒物业负责人处置（分无回款/有回款未结清两类），以便避免案件无限滞留。
- 验收标准：
  - **Given** 案件入池超过项目的 CFG-MAXCYCLE 且**无任何回款**，**When** 兜底任务运行，**Then** 提醒物业负责人（标注"无回款"），提供"结案/继续/发起法务"（BR-M8-08）。
  - **Given** 案件入池超过 CFG-MAXCYCLE 且**有过回款但仍未结清**，**When** 兜底任务运行，**Then** 同样提醒物业负责人（标注"有回款未结清"），提供同样三选项（BR-M8-08）。
  - **Given** 触发兜底提醒，**When** 物业负责人未处置，**Then** 案件**不自动结案**，保持"进行中"（BR-M8-08）。
  - **Given** 案件已结清，**When** 到达最长周期，**Then** 不提醒（已是终态）。

### Epic M8-D：结案后可见性收敛

**US-M8-06 结案后服务商/催收员只看统计**
- 作为**平台**，我想结案后让服务商/催收员只见统计、不见明细，以便保护隐私并符合隔离。
- 验收标准：
  - **Given** 案件已结案，**When** 服务商或催收员访问，**Then** **看不到案件明细**（业主信息/联系方式/跟进/录音/法务），**仅见统计聚合**（件数/回款/佣金/绩效）（BR-M8-09）。
  - **Given** 案件已结案，**When** 平台/物业按权限访问，**Then** 仍可见完整明细，用于质检/审计/对账/报表/存证（BR-M8-09/10）。
  - **Given** 案件已结案，**When** 系统处理公海/抢单/派单，**Then** 该案件不参与（BR-M8-10）。

## 5. 开放问题

- ~~OQ-M8-1：撤案/坏账是否需要平台审核~~ → **已定：不设审核流**。协调员/负责人可直接结案，平台超管仅代操作 + 事后可见（BR-M8-03/04）。
- ~~OQ-M8-2：撤案/坏账原因枚举~~ → **已定**：平台超管可配（CFG-CLOSE-REASONS），"其他"须填备注（BR-M8-05）。
- ~~OQ-M8-3：隐私窗口~~ → **已定**：结案即对服务商/催收员收敛为"只看统计、不看明细"；明细仅平台+物业可见，**无 30 天过渡**（BR-M8-09）。
- ~~OQ-M8-4：分期到账判定/逾期处理~~ → **已定**：①结清=政策分期**各期累计＝减免后应收**；②**(a) 逾期分期不特殊处理**——案件仍"进行中"继续催，靠承诺跟踪/最长催收周期兜底（不作废分期、不强制恢复全额）。
- ~~OQ-M8-5：结案重开~~ → **已定：(a) 自动重开**——冲正/退款致未结清，案件**自动回退为"进行中"**（已计佣金同步冲减，BR-M9-06）。
- ~~OQ-M8-6：业主超额/多缴~~ → **已定：系统不存在多缴场景**——本期回款由协调员**线下按实际金额标记**（多缴线下处理）；在线缴费（后期）**只能按应缴金额缴**，无法多缴。故系统无需多缴处理。
- ~~OQ-M8-7：分期跨催收员佣金归属~~ → **已定**：每笔回款的付佣归**该笔到账时点的持有服务商**（M9 BR-M9-04）。
