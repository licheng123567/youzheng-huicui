# 有证慧催 PRD · M10 经营报表

> 依赖：[00-总纲](./00-总纲.md) §5 导航主轴、[M2](./02-项目批次案件主数据.md)（项目/批次/区域聚合）、[M8](./08-结案与生命周期.md)（结案统计）、[M9](./09-结算与对账.md)（佣金/计费）。
> **契约对齐**：本模块与 API 契约 v1.0.0 对齐。
> 目标：定义各角色的经营报表——**平台收入+支出+利润**、物业催收效果、服务商业绩，按**项目→批次**聚合下钻、按区域/周期维度，严格隔离。

---

## 1. 模块范围

- **平台报表**：利润 =（佣金收入 − 佣金支出）+ 能力收入（**本期不含能力成本**，BR-M10-01），按项目/批次/区域/周期聚合下钻。
- **物业报表**：回款/回款率、催收进度、佣金支出、撤案分布、能力计费（分钟/短信/存证）。
- **服务商报表**：承接量/回款业绩/付佣收入、催收员产能、分钟消耗、释放/撤案监控。
- 角色化、隔离（物业↔服务商互不可见对方经营数据）。

**不在本模块**：结算计算本身（M9）、对账单（M9）、案件明细（M2）。

---

## 2. 关键概念

| 概念 | 说明 |
|---|---|
| 聚合主轴 | 项目(小区) → 批次(催收单) → 案件；报表可按项目聚合下钻批次，或按单批次出（总纲 §5） |
| 区域维度 | 项目区域属性（M2 BR-M2-09），平台报表可按区域聚合 |
| 平台利润 | 收入(物业佣金) − 支出(服务商佣金 + STT/短信/存证通道成本等) |
| 回款率 | 回款额 / 应收（减免后），M2 聚合指标 |

---

## 3. 业务规则

- **BR-M10-01（平台损益）**：**本期**平台利润 = **（佣金收入 − 佣金支出）+ 能力收入**（**能力成本本期不计入**，留后期）。
  - **佣金收入** = 物业佣金（M9 收佣）；**佣金支出** = 服务商佣金（M9 付佣）。
  - **能力收入** = 向物业收取的转写费 + 短信费 + 存证费 + 法律服务费（M9）。
  - **能力成本**（付第三方通道/律师）= **后期纳入**（届时利润再减此项）。
  - 可按**项目/批次/区域/周期**聚合下钻（BR-M2-07/09）。
- **BR-M10-02（按批次/项目聚合下钻）**：报表沿用导航主轴，支持按项目聚合并下钻批次、或独立按单批次出（总纲 §5）。
- **BR-M10-03（角色化与隔离）**：
  - 物业只见**本物业**经营数据（回款/进度/佣金支出/能力计费/撤案分布）；
  - 服务商只见**本服务商**经营数据（承接/回款业绩/付佣收入/催收员产能/分钟消耗/释放·撤案监控）；
  - **物业↔服务商互不可见对方经营数据**；平台见全平台。
- **BR-M10-04（指标一致）**：报表指标与 M2 聚合口径一致（案件数/应收/回款/回款率/已结清/在催/法务处理数等）；统计随数据实时/准实时更新（刷新口径 TBD）。
- **BR-M10-05（监控类指标）**：报表纳入**释放频次**（M3 BR-M3-27）、**撤案频次/原因分布**（M8 BR-M8-12）等防滥用监控维度（对应角色可见）。
- **BR-M10-06（平台员工数据范围）**：平台员工看报表受其**数据范围**约束（M1 BR-M1-14，区域/服务商/物业），范围外不计入其报表。
- **BR-M10-07（结案数据）**：结案/归档案件计入统计聚合；服务商侧报表只见统计、不见明细（M8 BR-M8-09）。
- **BR-M10-08（导出）**：报表支持导出（格式/字段 TBD），导出受同样隔离与权限约束；**每次导出留痕**（操作人/时间/导出内容范围），供数据安全审计。
- **BR-M10-09（考核层级）**：**催收员个人产能/排行仅服务商内部可见**（服务商自管自看）；**平台报表只考核到服务商层级、不下钻到催收员个人**。与 M3 去激励化一致（产能仅统计，不做平台级个人排名）。
- **BR-M10-10（成员工作督导 = 谁的员工谁督导）**：组织负责人对**本组织成员**有「工作督导」——按成员统计本月**工作量/质量**（处理案件数 / 关键产出 / 平均响应时长 / 关联质检风险数 / **超时未处理数**），对异常成员可发起**督导处理（提醒 / 督导谈话 / 安排培训 / 记录，对应 SupervisionAction: REMIND / TALK / TRAINING / NOTE）并留痕**。**物业负责人督导协调员、服务商负责人督导催收员**（与 BR-M5-07a「谁的员工谁处理」一致），**仅督导本组织员工**。督导**不等同处罚**——警告/停权/调整走成员管理权限与组织内部流程；个人产能口径仍受 BR-M10-09 约束（不对平台/他组织开放）。
- **BR-M10-10（物业看服务商表现，仅展示）**：物业**可查看"服务于本物业案件"的服务商表现**（如本物业项目上的回款率、时效等）；**仅限本物业范围**，不泄露该服务商在其他物业的经营数据（隔离）。**纯报表展示，不含评价、不影响派单**（本产品不做物业评价服务商）。

## 4. Epic 与用户故事

### Epic M10-A：平台经营报表

**US-M10-01 平台收入+支出+利润报表**
- 作为**平台超管/有权员工**，我想看收入支出利润并按项目/批次/区域下钻，以便经营决策。
- 验收标准：
  - **Given** 有结算数据，**When** 我查看平台报表，**Then** 见利润=**（佣金收入−佣金支出）+ 能力收入**（本期不含能力成本），可按项目/批次/区域/周期聚合下钻（BR-M10-01/02）。
  - **Given** 我是受限数据范围的平台员工，**When** 我查看报表，**Then** 仅含我数据范围内对象（BR-M10-06）。

### Epic M10-B：物业报表

**US-M10-02 物业催收效果报表**
- 作为**物业负责人**，我想看本物业回款/进度/佣金支出/撤案分布/能力计费，以便评估催收成效与成本。
- 验收标准：
  - **Given** 本物业有数据，**When** 我查看报表，**Then** 见回款额/回款率/进度/已结清/佣金支出/撤案频次分布，按项目/批次下钻（BR-M10-01/03/05）。
  - **Given** 查看「能力用量汇总」（STT/短信/存证/法律服务），**Then** **仅展示用量（分钟/条/次）、不展示金额**（与计费明细一致 BR-M9-06b）；金额以对账单为准。
  - **Given** 进入工作台，**Then** 顶部按 **今日 / 本月 / 自定义时段** 统计经营指标（回款金额、回款案件数、联系案件数、通话分钟数、处理工单数、短信发送数、存证次数等），并见**按天/按月回款趋势**。
  - **Given** 多个服务商催过本物业的案子，**When** 我查看，**Then** 见**各服务商在本物业项目上的表现对比**（回款率/时效/风险率），**仅限本物业范围**，不含其在他物业的数据（BR-M10-10）。
  - **Given** 我是物业，**When** 查看，**Then** 不可见任何服务商付佣/其他物业数据（BR-M10-03）。

### Epic M10-C：服务商报表

**US-M10-03 服务商业绩报表**
- 作为**服务商负责人**，我想看承接量/回款业绩/付佣收入/催收员产能/分钟消耗/释放监控，以便管理团队。
- 验收标准：
  - **Given** 本服务商有数据，**When** 我查看报表，**Then** 见承接量/回款/付佣收入/各催收员产能/分钟消耗/释放频次，按项目/批次下钻（BR-M10-01/03/05）。
  - **Given** 案件已结案，**When** 服务商查看，**Then** 只见统计聚合、不见案件明细（BR-M10-07、M8 BR-M8-09）。
  - **Given** 我是服务商，**When** 查看，**Then** 不可见物业收佣/其他服务商数据（BR-M10-03）。

### Epic M10-D：导出

**US-M10-04 报表导出**
- 作为**有权角色**，我想导出报表，以便线下分析/留档。
- 验收标准：
  - **Given** 我有报表权限，**When** 我导出，**Then** 导出内容受同样隔离/数据范围约束（BR-M10-08/06）。
  - **Given** 我导出报表，**When** 导出完成，**Then** 系统**留痕**（操作人/时间/范围），供安全审计（BR-M10-08）。
