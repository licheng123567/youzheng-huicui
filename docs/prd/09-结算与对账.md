# 有证慧催 PRD · M9 结算与对账

> 依赖：[00-总纲](./00-总纲.md) §2 商业模式、§8 CFG、[M2](./02-项目批次案件主数据.md)（批次佣金比例）、[M4](./04-催收作业.md)（短信通道/分钟）、[M8](./08-结案与生命周期.md)（回款/分期）。
> **契约对齐**：本模块与 API 契约 v1.0.0 对齐。
> 目标：定义两条佣金结算线、能力计费（预充：通话分钟 / 短信条数；按次计费：存证 / 法律服务）、以及平台↔物业、平台↔服务商对账。强约束：**物业 ↔ 服务商 无任何资金往来**。

---

## 1. 模块范围

- **佣金结算**：平台 ← 物业（收佣，按批次比例）；平台 → 服务商（付佣，按服务商×批次比例）。
- **能力计费**：**预充扣减**——通话解析分钟数（物业/服务商）、物业短信条数；**按次计费、计入对账**——存证、法律服务（均只向物业）。
- **对账**：平台↔物业、平台↔服务商，分别对账，互不互通。
- **分期/跨催收员的佣金归属**口径。

**不在本模块**：回款登记（M4/M7）、报表展示（M10）、结案判定（M8）。

---

## 2. 关键概念

| 概念 | 说明 |
|---|---|
| 收佣（平台←物业） | 佣金 = 回款 × 收佣比例（按批次，CFG-COMM-IN） |
| 付佣（平台→服务商） | 佣金 = 回款 × 付佣比例（按服务商×批次，CFG-COMM-OUT） |
| 分钟充值/扣减 | 通话解析分钟数预充、按解析扣减（物业/服务商各自账户） |
| 短信条数 | 物业短信通道按条计费，平台给物业充值（CFG-SMS-COUNT） |
| 存证/法律服务计费 | 只向物业收费（CFG-EVIDENCE-PRICE） |
| 支付申请单 | 收款方按案件回款明细手动勾选生成的结算单据（取代原"对账单"）；收佣线=平台生成→物业付款→平台确认收款；付佣线=服务商生成→平台付款+凭证（BR-M9-12a/b）|

## 3. 业务规则

### 3.1 佣金结算
- **BR-M9-01（收佣）**：平台向物业收佣 = **回款 × 收佣比例**，比例**按批次**（CFG-COMM-IN）；以**实际到账回款**为计算基数（H5/线下合并，M8 BR-M8-02）。
- **BR-M9-01a（收佣比例录入时点）**：收佣比例（平台↔物业协商）**在创建项目时录入项目级默认值**，并在**批次导入时按批次设定/覆盖**（项目级为默认、批次级优先）；为必填项，避免"批次无约定佣金比例"。付佣比例（平台→服务商）在**平台派单/开放抢单**时设定（BR-M9-18 / 派单）。比例均受防倒挂护栏（BR-M9-14：付佣≤收佣）。
- **BR-M9-01b（佣金基数口径）**：佣金基数 = **减免后实收的实际回款、不含税**；**部分回款按每笔实际到账即时 × 比例**计佣（非等结清后一次算）。收佣、付佣同此口径。展示层应标注"基数=减免后实收、不含税"（避免误解为含税基数）。
- **BR-M9-15（结算时机 = 按案件明细手动组支付申请单，BR-M9-12a）**：佣金随回款**逐笔计佣**（即时×比例，BR-M9-01b）、逐笔记录到账时点与持有服务商（付佣归属，BR-M9-04）；**结算 = 收款方勾选未结回款明细生成支付申请单**（收佣线平台生成、付佣线服务商生成），**手动组单、非按月自动**；完成前可撤销/撤回重组（BR-M9-12b/12d）。**替代原"批次结清一次性"与"按月自动增量对账"两套旧口径。**
- **BR-M9-02（付佣）**：平台向服务商付佣 = **回款 × 付佣比例**，比例**按服务商 × 批次**（CFG-COMM-OUT）。
- **BR-M9-03（归属）**：付佣归属**回款发生时该案件的持有服务商**（开放抢单案件归抢中者所属服务商，M3 BR-M3-15）。
- **BR-M9-19（服务商内部催收员佣金）**：平台只与服务商结算（BR-M9-02），**不核算催收员个人提成**；但系统为**服务商内部**提供催收员佣金功能——服务商可**给催收员设佣金比例**，系统按该催收员的回款业绩**自动计算其应得佣金**，服务商可**逐笔/逐人标记是否已支付**。此结算为服务商内部使用，平台不参与、不对账。
- **BR-M9-19a（催收员"我的结算/佣金"自查视图）**：催收员可查看**本人**的结算/佣金明细——按**批次 / 周期**列出我的回款 → 提成（比例 × 回款）、累计与**已结 / 待结**状态，可导出。**这是服务商内部个人提成结算，不是平台↔组织对账**（与 BR-M9-02 付佣对账口径相互独立、不得混淆）；提成比例与是否已支付由服务商按 BR-M9-19 设定/标记，**催收员只读**。结案后按 M8 BR-M8-09 收敛为统计口径。分期回款**逐笔按到账时点结算**；案件中途释放/换持有人时，**每笔回款的付佣归该笔到账时点的持有服务商**（解 OQ-M8-7）。催收员个人佣金见 BR-M9-19（服务商内部）。
- **BR-M9-05（撤案/坏账无佣金）**：撤案/坏账（无回款关闭）**不产生佣金**；仅已回款部分按比例已结算的不冲回（除非回款被冲正，见 BR-M9-06）。
- **BR-M9-06（回款冲正联动）**：线下回款冲正/退款（M4 BR-M4-07）→ 相应**冲减已计佣金**（收佣与付佣同步冲减），留痕。
- **BR-M9-18（开放抢单付佣比例）**：平台标记某批次案件"开放抢单"**前，必须先为该批次设置"开放抢单付佣比例"**（未设不可开放）；任一服务商抢中后**按该比例结算付佣**（不依赖 (服务商×批次) 预配）。该比例同样受防倒挂护栏约束（BR-M9-14）。
- **BR-M9-14（防倒挂护栏）**：同一批次的**付佣比例不得高于收佣比例**（平台净收 = 收佣 − 付佣 ≥ 0）。配置付佣比例时**系统校验**，若 > 该批次收佣比例则**拒绝并提示倒挂**。视角说明：收佣是"物业给平台"（如回款×30%），付佣是"平台给服务商"（如回款×20%），两者独立配置但受此护栏约束。

### 3.2 能力计费与充值
> **计量口径（不搞余额制/能力账户）**：通话解析按**分钟数**、短信按**条数**、存证按**次**、法律服务按**次**，各算各的、分别计量。
- **BR-M9-06a（充值由平台后台·本阶段不开放自助）**：本阶段**不开放物业/服务商自助充值**——分钟 / 短信 / 存证 / 法律服务 次数均**由平台超管后台充值**（代充，M1 BR-M1-15）；充值中心对物业/服务商**只读展示余额与本月用量**，充值入口仅平台可见。后续如开放自助充值再行调整。
- **BR-M9-06b（计费明细 = 月→日→明细下钻·仅用量）**：计费明细**只记用量、不展示金额**；按**月汇总 → 日汇总 → 单笔明细**三级下钻，单笔明细含 业主/房号/项目/批次/类型/用量/状态，便于核对到户。
- **BR-M9-07（通话解析分钟，预付费）**：通话解析分钟为**预付费**（先充后用），物业与服务商各自**预充分钟数**、按分钟扣减（M5）；分钟数低于阈值预警（CFG-MIN-WARN）/ 解析暂停（CFG-MIN-STOP）。**平台超管可代充值**（M1 BR-M1-15）。
- **BR-M9-08（短信条数，预付费）**：物业短信为**预付费**，**预充条数**（CFG-SMS-COUNT）、按条扣减；条数低于阈值预警（CFG-SMS-WARN）；物业可查**发送记录/明细**（M4 BR-M4-16）。
  - **发送回执**：成功/失败/未达，**依短信通道能力尽量呈现**；
  - **失败不退条数**，但**记录失败原因并给出汇总**（如空号/停机/黑名单/内容拦截等），让物业了解失败构成。
- **BR-M9-09（短信通道）**：**短信签名与模板均由平台统一配置（CFG-SMS-SIGN/TPL），物业不创建、无审核**；物业只充值条数、用平台配好的发缴费短信（M4 BR-M4-16）。
- **BR-M9-10（存证/法律服务计费）**：存证与法律服务**只向物业收费**，服务商不计费、不可见。两者计费方式不同：
  - **存证**：**按次计费、三类场景同一单价**（CFG-EVIDENCE-PRICE，M6 BR-M6-03）。
  - **法律服务**（催收函/律师函/起诉状等）：**按次计费、计入对账**；单价由平台↔物业协商（可按文书类型不同），**不搞预存余额**（CFG-LEGAL-FEE）。

### 3.3 对账
- **BR-M9-11（双线对账隔离）**：**平台↔物业**对账（收佣 + 物业侧计费：分钟/短信/存证/法律服务）与**平台↔服务商**对账（付佣 + 服务商侧分钟计费）**完全独立、互不可见**；**物业 ↔ 服务商 无任何资金/对账接口**（强约束）。
- **BR-M9-12（支付申请单）**：支付申请单含所含回款明细、应结佣金、计费明细。**结算口径见 BR-M9-12a（按案件明细手动组单 + 撤销/撤回重生成，替代原"批次结清一次性"与"按月自动对账"）**。
  - **支付构成可下钻**：支付申请单可查看**支付明细构成**（回款基数 × 比例 = 应结、**逐案件回款明细**、能力计费构成）。
  - **纠错 = 撤销/撤回重生成**（取代原"确认/驳回/申诉 + 案件级异议"）：完成前生成方可撤销/撤回 → 改明细 → 重新生成（BR-M9-12c 作废、12d）。
  - 支付申请单**可下载**、**支持盖电子章**（电子签章方案 TBD）。
- **BR-M9-12a（支付申请单 = 按案件明细手动组单，替代"按月自动对账"）**：
  - **结算单元 = 回款明细行（案件 × 缴款）**，每笔状态 `未结算 → 已纳入支付申请单 → 已结算(锁定)`。已锁定明细不再被重复纳入。
  - **手动勾明细组单、不按月自动**：收款方**勾选案件回款明细**生成一张**支付申请单（PaymentRequest）**，可跨期、可分多单；批次结案时对剩余未结明细**提示组尾单（不自动出单）**。
  - **对账列表仍按批次汇总**（回款基数/笔数/回款率/比例/应结/已结/未结，操作「回款明细」展示 + 「支付申请单列表」），但**出单由"按月自动"改为"勾明细手动"**。
  - **取代** 原"按月自动增量对账 + 对账单确认/驳回/申诉" 与 "案件级在线异议"（后者作废，见 BR-M9-12c）——纠错统一走"撤销/撤回 → 改明细 → 重新生成"。
- **BR-M9-12b（双线对称：收款方生成、付款方付款、生成方完成前可撤）**：
  - **收佣线（物业付平台佣金）**：**平台**勾选明细生成支付申请单 → 物业查看并线下付款 → **平台确认收款（+收款凭证）= 完成锁定**。**平台确认收款前可「撤销」** → 单作废、明细释放、可重新组单。物业**只查看**（不确认/不驳回）。
  - **付佣线（平台付服务商佣金）**：**服务商**勾选明细生成支付申请单 → 平台付款 → **平台上传支付凭证 = 完成锁定**。**平台支付前服务商可「撤回」** → 单作废、重新组单。
  - **状态机**：`PENDING(待付·可撤销/撤回) → PAID(已完成·凭证·锁定)`；撤销/撤回 → `VOIDED(作废)`，所含明细回未结算。
  - 佣金 = 减免后实收回款（不含税 BR-M9-01b）× 合同约定比例（防倒挂 BR-M9-14）；单金额 = Σ(明细回款 × 比例)，比例不在结算中改。
- **BR-M9-12c（案件级在线异议）—— 已作废**：原"单笔回款在线异议（异议中/平台核对/调明细重算/冲正）"流程**取消**（线上异议易纠缠、不好处理）。**改为**：对有争议明细，由**生成方撤销/撤回支付申请单 → 剔除或修正该笔明细 → 重新生成**；争议沟通走线下，系统只留撤销/重生成痕迹。
- **BR-M9-12d（撤销/撤回 + 凭证 + 单据能力）**：
  - **撤销（收佣线·平台）/ 撤回（付佣线·服务商）**：仅 `PENDING`（对方未付款）时可做；执行后 `VOIDED`、明细释放、可重组；留痕（操作人/时间/原因可选）。
  - **完成=凭证**：收佣线平台**确认收款**留**收款凭证**；付佣线平台**上传支付凭证**。完成即 `PAID` 锁定、明细 `已结算`，不可再撤（纠错走线下）。
  - 支付申请单**可打印/下载/盖电子章/发送对方**，下钻见所含回款明细（案件/批次/金额），可追溯。
- **BR-M9-16（支付申请单据 + 凭证）**：支付申请单即结算单据（收佣线=平台向物业收的收款单、付佣线=平台付服务商的付款单），含所含回款明细；完成时留收款/支付凭证（BR-M9-12d）；**可打印 / 下载 / 盖电子章 / 发送对方**。
  - 单据**关联其覆盖的回款明细**：可下钻查看"本次结算对应哪些案件/批次的哪些回款"，金额可追溯。
  - 实际打款后**上传打款/收款凭证**，关联该结算单据留存，供审计。
- **BR-M9-13（凭证号）**：对账以**批次号**（M2 BR-M2-02 可读编码）为主要凭证维度，可按项目/周期聚合。

## 4. Epic 与用户故事

### Epic M9-A：佣金结算

**US-M9-01 平台对物业收佣结算**
- 作为**平台**，我想按批次比例对物业回款收佣，以便实现营收。
- 验收标准：
  - **Given** 案件有回款，**When** 逐笔计佣，**Then** 收佣 = 每笔减免后实收(不含税) × 生效收佣比例（即时×比例，BR-M9-01b/15）。
  - **Given** 要收佣，**When** 出单结算，**Then** 走支付申请单（平台勾明细生成→物业付款→平台确认收款，US-M9-06）；**非按月自动**。

**US-M9-02 平台对服务商付佣结算**
- 作为**平台**，我想按服务商×批次比例对回款付佣，以便结算服务商收入。
- 验收标准：
  - **Given** 服务商持有案件有回款，**When** 逐笔计佣，**Then** 付佣 = 每笔实收 × 该(服务商×批次)付佣比例（BR-M9-01b/15）。
  - **Given** 要付佣，**When** 出单结算，**Then** 走支付申请单（服务商勾明细生成→平台付款+凭证，US-M9-07）；**非按月自动**。
  - **Given** 分期回款跨持有人，**When** 每笔到账，**Then** 该笔付佣归到账时点的持有服务商（BR-M9-04）。
  - **Given** 开放抢单案件回款，**When** 结算，**Then** 付佣归抢中者所属服务商（BR-M9-03）。
  - **Given** 我配置某(服务商×批次)付佣比例**高于该批次收佣比例**，**When** 提交，**Then** 系统**拒绝并提示倒挂**（BR-M9-14）。

**US-M9-09 催收员"我的结算/佣金"自查**
- 作为**催收员**，我想查看自己的结算/佣金明细，以便核对个人提成。
- 验收标准：
  - **Given** 服务商已为我设佣金比例，**When** 我打开"我的结算"，**Then** 按批次/周期看到我的回款 → 提成、累计与已结/待结状态，可导出（BR-M9-19a）。
  - **Given** 我查看，**When** 浏览明细，**Then** 仅**只读**（比例设定/是否已支付由服务商操作，BR-M9-19）；该视图为**服务商内部提成**，**不等同于平台↔服务商付佣对账**（BR-M9-02）。

**US-M9-10 服务商设催收员佣金并按明细支付**
- 作为**服务商负责人**，我想给催收员设佣金比例、按批次/案件明细生成支付单据并标记支付，以便内部提成结算（平台不参与）。
- 验收标准：
  - **Given** 催收员承接多个批次，**When** 我维护佣金，**Then** 主表按**催收员聚合**、可穿透到**各批次（比例可不同）→ 案件回款明细**；系统按回款业绩自动算应得佣金（BR-M9-19）。
  - **Given** 比例设定，**When** 我设催收员某批比例，**Then** 不得超过该（服务商×批次）**付佣比例**（防倒挂，US-M9-02），超出禁止保存。
  - **Given** 有未结明细，**When** 我勾选并**生成支付单据**，**Then** 生成待支付结算单；在**结算单列表**「确认支付完成」即结算完成，对应明细按笔锁定（BR-M9-19）。
  - **Given** 该结算为服务商内部，**When** 操作，**Then** 平台不参与、不对账，物业侧不可见付佣线（BR-M9-02/11）。

### Epic M9-B：能力计费与充值

**US-M9-03 通话解析分钟充值与扣减**
- 作为**物业/服务商负责人**，我想充值并查看分钟消耗，以便不中断解析。
- 验收标准：
  - **Given** 我充值通话解析分钟，**When** 充值成功，**Then** 本组织余额增加，并生成**充值流水**；可查**充值/扣减全量流水**（时间/金额/分钟/操作人/关联通话）（BR-M9-07）。

**US-M9-04 物业短信条数充值与明细**

- 验收标准：
  - **Given** 我充值短信条数，**When** 成功，**Then** 条数增加、可查扣减明细（BR-M9-08）。
  - **Given** 我发缴费短信，**When** 发送，**Then** 用平台统一配置的签名+模板发出（无需我配置/审核）（BR-M9-09）。
  - **Given** 短信已发送，**When** 我查看，**Then** 见发送记录/状态（成功/失败/未达，依通道能力）/明细（BR-M9-08）。

### Epic M9-C：对账与结算单据

**US-M9-06 收佣线支付申请单（平台 ← 物业）**
- 作为**平台**，我想勾选案件回款明细向物业生成支付申请单、收款后确认，以便完成收佣结算。
- 验收标准：
  - **Given** 某批次有未结回款明细，**When** 平台**勾选明细生成支付申请单**，**Then** 单含所含回款明细 + 应结收佣（基数×收佣比例）+ 计费明细，状态 `PENDING`（BR-M9-12a/b）。

  - **Given** 我是物业，**When** 查看，**Then** **只查看、不确认/不驳回/不异议**；疑问走线下（取消在线异议 BR-M9-12c）；不可见付佣数据（BR-M9-11）。

**US-M9-07 付佣线支付申请单（平台 → 服务商）**
- 作为**服务商负责人**，我想勾选案件回款明细向平台生成支付申请单、平台付款后凭证留存，以便完成付佣结算。
- 验收标准：
  - **Given** 本商承接批次有未结回款明细，**When** 服务商**勾选明细生成支付申请单**，**Then** 单含明细 + 应结付佣（基数×付佣比例），状态 `PENDING`（BR-M9-12a/b）。
  - **Given** 单为 `PENDING`、平台尚未支付，**When** 服务商**撤回**该单，**Then** 单 `VOIDED`、明细释放、可重新组单（BR-M9-12d）。
  - **Given** 平台付佣完成，**When** 平台**上传支付凭证**，**Then** 单 `PAID` 锁定、明细 `已结算`，支付结束（BR-M9-12b/d）。
  - **Given** 我是服务商，**When** 查看，**Then** 不可见物业侧收佣/存证计费（BR-M9-11）。

**US-M9-08 支付申请单据与凭证**

- 验收标准：
  - **Given** 一张支付申请单，**When** 我下载，**Then** 可**打印/下载/盖电子章/发送对方**（BR-M9-16）。

  - **Given** 完成支付/收款，**When** 上传支付/收款凭证，**Then** 凭证关联该单留存供审计（BR-M9-12d/16）。
  - **Given** 单据属某一方，**When** 另一方查看，**Then** 受双线对账隔离约束，不可见对方单据（BR-M9-11）。
