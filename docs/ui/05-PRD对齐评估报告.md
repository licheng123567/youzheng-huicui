## 3. 关键差异清单（按严重度排序）

> 仅列真实缺失 / 不一致，区分"原型可接受占位"。每条含 PRD 依据 / UI 现状 / 建议。

### 🔴 高严重度（影响核心闭环、隔离正确性、安全、合规可演示性）

**H-1 组织管理视图完全缺失（建组织 = 多租户的源头）**
- PRD 依据：US-M1-01/02、BR-M1-01/02（平台开通物业/服务商组织，设唯一负责人账号）。
- UI 现状：index.html navMeta 无 orgMgmt；SA nav（L947）无组织管理入口；成员管理（view=members）混用"平台组织管理"与"本组织员工"，仅能管当前组织员工。
- 建议：SA nav 新增「组织管理」，实现组织列表（平台/物业/服务商）+ 新建组织表单（名称+负责人账号名+手机号）+ 组织详情含成员入口。可复用 members 结构快速搭建。

**H-2 系统级操作日志页缺失（审计留痕无独立视图）**
- PRD 依据：BR-M1-08（授权变更/账号启停/登录/负责人交接均留操作日志）、BR-M1-15（代操作被代方可见）。
- UI 现状：时间线有单条案件级代操作样例（L1005）+ oplog 抽屉（L898），但无全局/组织级审计日志视图。
- 建议：SA/PL/VL nav 新增「操作日志」，全局审计表（时间/操作人/动作/代操作标注），按角色范围过滤，被代方可见以验证 BR-M1-15 透明性。

**H-3 SE 平台员工数据范围过滤未落地（与 BR-M1-14 冲突）**
- PRD 依据：BR-M1-14（平台员工功能+数据范围双维度授权，范围外对象不可见）。
- UI 现状：复核 visibleCases（L1027-1034）确认——SE scope='all'，无任何过滤分支，scopeLabel 仅文案"数据范围内(区域/服务商/物业)"。SE 切换后看到全平台数据，与隔离原则冲突。
- 建议：roles.SE 增加 dataRange 字段（如 {areas:['沪/浦东'],providers:['安信催收']}），visibleCases 增加 SE 范围过滤分支，按 area/provider 匹配。

**H-4 协调员多对多关联入口缺失（直接影响协调员可见案件）**
- PRD 依据：US-M2-02、BR-M2-13（协调员关联到项目/批次，多对多）。
- UI 现状：项目档案、批次详情、成员管理均无协调员关联管理入口。
- 建议：项目档案（selProj）/批次详情增加「协调员」分区，PL 可添加/移除（can('proj.edit')），账号来源本物业 members。

**H-5 多联系方式管理完全缺失（催收效率核心场景）**
- PRD 依据：US-M4-02、BR-M4-11（号码标签/主号码/无效标注，增改留痕）。
- UI 现状：案件详情左栏仅单一 phone 字段；App 拨号无多号码选择。
- 建议：案件详情「欠费详情」下增加「联系方式」区块（号码+标签+主号码徽标+无效状态+设主/标无效操作），App 拨号页支持多号码选择。

**H-6 主动释放原因弹窗 + 释放频次查看缺失**
- PRD 依据：BR-M3-27（主动释放须填原因且留记录，VL 可见频次）。
- UI 现状：case.release 触发仅 toast，无原因枚举弹窗（无法联系/非本人/其他）；VL 无释放历史/频次入口。
- 建议：case.release 增加弹窗（原因枚举下拉+备注+提交，无原因阻止）；VL「我的团队」增加「释放记录」tab。

**H-7 公海实时竞争态缺失（US-M3-09 核心）**
- PRD 依据：BR-M3-20/21/22（入池来源徽标/正在查看 N 人/实时事件日志，不设动态规则）。
- UI 现状：providerSea/platformSea 列表无实时人数、无入池来源徽标、无事件日志面板。
- 建议：列表增加入池来源徽标列（T1 超时/T2 退回/开放抢单）+「正在查看 N 人」气泡，页面右侧增加实时事件日志流水面板。

**H-8 服务商内海池辅助缺失（T2 倒计时 + 余量条 + 智能推荐）**
- PRD 依据：BR-M3-23（US-M3-05/08 核心验收）。
- UI 现状：服务商公海无 T2 倒计时列；VL「分配」仅 toast，无催收员持仓余量面板/推荐标注。
- 建议：公海列表加「T2 剩余」列；分配弹窗展示本商催收员列表+持仓余量进度条+「推荐」标注。

**H-9 余额不足后手动/批量补解析入口缺失（BR-M5-02 核心场景）**
- PRD 依据：US-M5-02、BR-M5-02（充值后手动补解析，支持批量按时间顺序到余额扣完）。
- UI 现状：有余额暂停提示，但无「待解析录音列表」+「批量补解析」动作。
- 建议：充值中心/计费明细增加「待解析录音」列表（录音时间/时长/状态）+「批量补解析」按钮+进度反馈。

**H-10 录音存证前置条件校验缺失（违反 BR-M6-02 业务规则）**
- PRD 依据：BR-M6-02（仅发起法务后协调员才能对录音存证）。
- UI 现状：发起存证入口宽松，未校验案件是否已进入法务，可对任意录音发起录音存证。
- 建议：发起录音存证时校验案件法务状态，未进入法务则禁用并提示「请先发起法务再对录音存证」。

**H-11 App 端结案收敛缺失（隔离风险，非占位）**
- PRD 依据：BR-M8-09（VL/CO 结案后仅见统计，业主信息脱敏，立即生效）。
- UI 现状：app.html 无 redactClosed 逻辑，结案案件仍以普通案件展示、业主信息未脱敏。
- 建议：app.html 案件列表/详情增加结案检测，对 CO 显示脱敏统计视图（件数/回款额），隐藏联系方式/跟进明细。

**H-12 H5 令牌机制 + 过期态完全缺失**
- PRD 依据：BR-M7-01（短信/微信链接携令牌直达，OQ-M7-2 防遍历/防分享滥用）。
- UI 现状：复核 h5.html 确认——全文无 `<script>`、无 URLSearchParams、无 token；业主信息硬编码；分期仅 HTML 注释占位。
- 建议：h5.html 加 token 参数解析（?token=xxx）+ expired 状态视图（失效时仅显示"链接已失效，请联系物业重发"，不渲染账单），使越权隔离可演示。

**H-13 结算单据生成/下钻/打款凭证上传缺失（BR-M9-16）**
- PRD 依据：BR-M9-16（批次结清生成独立结算单据、关联回款明细下钻、上传打款凭证）。
- UI 现状：reconIn/reconOut 仅对账列表，无结算单据详情/下钻/凭证上传。
- 建议：对「已确认」对账单增加「生成结算单据」→ 单据详情页（金额/回款明细下钻/凭证上传/下载/电子章，P-BILL-03）。

**H-14 防倒挂护栏缺失（BR-M9-14）**
- PRD 依据：BR-M9-14（付佣比例不得高于收佣比例，系统校验拒绝）。
- UI 现状：无任何倒挂校验交互。
- 建议：批次付佣比例编辑处实时校验，≥收佣比例时红色错误提示+禁用保存。

**H-15 经营报表纯占位，三角色内容均缺 + 无下钻（BR-M10-01/02/03）**
- PRD 依据：US-M10-01/02/03、BR-M10-02（项目→批次聚合下钻，三角色隔离内容）。
- UI 现状：复核 reports 视图（L720-739）确认——仅 4 张 KPI 卡 + alert 占位 + 静态"待统计/—"，切换角色仅 scopeLabel 文字变化，无内容差异、无下钻。
- 建议：reports 按 role 渲染三套内容（SA/SE 平台损益、PL 物业视角、VL 服务商视角）+ 项目→批次下钻表格。
- 备注：M10 属 PRD 外围模块，占位形态可部分接受，但"三角色内容零差异"是真实未对齐，至少应做角色化 KPI 内容区分。

**H-16 高危操作无二次确认弹窗（防误触）**
- PRD 依据：撤案/坏账（BR-M8-03/05）、停用（BR-M1-07）、冲正（BR-M9-06）均为不可逆/高影响操作。
- UI 现状：onOp 对 case.close/case.reverse/停用直接 toast，无 dialog。
- 建议：复用已有 .dialog 组件，为 3 类高危操作增加前置二次确认（含"不可撤销"警示）。

**H-17 全平台零无障碍（ARIA / label / 焦点管理）**

## 5. 结论与下一步建议（改造项排序）

**总评：原型在"业务核心闭环 + 多租户/资金双线隔离 + 结案脱敏 + 页面复用 CRM 理念"上达成度高（这是最难、最体现产品价值的部分，已构成可演示的护城河），失分集中在"管理源头（M1）+ 交易结算与报表（M9/M10）+ 三端安全收尾（H5 令牌、App 脱敏）+ 系统性可用性（无障碍/二次确认）"。** 这意味着原型已能讲清楚"产品怎么跑通"，但尚不足以讲清楚"怎么开通、怎么算钱、怎么经营复盘、怎么防误操作/越权"。

### 推荐改造批次（按 ROI + 风险排序）

**第一批（上线前必修 / 隔离与安全，1–2 天）**
1. App 端结案脱敏收敛（H-11）+ H5 令牌与过期态（H-12）——堵隔离/越权风险。
2. SE 数据范围过滤落地（H-3）+ VL 加 qc/projects（M-4/M-9）——补隔离一致性。
3. 高危操作二次确认（H-16）+ 防倒挂护栏（H-14）+ 录音存证前置校验（H-10）——防误操作/违规。

**第二批（源头管理与交易完整性，2–3 天）**
4. 组织管理视图（H-1）+ 系统操作日志页（H-2）——补 M1 源头与审计。
5. 结算单据生成/下钻/凭证（H-13）+ SE 财务导航（M-14）+ 充值流水（M-15）——补 M9 交易闭环。
6. 主动释放原因弹窗（H-6）+ 多联系方式管理（H-5）+ 补解析入口（H-9）——补高频作业缺口。

**第三批（决策辅助与经营复盘，3–4 天）**
7. 公海实时竞争态（H-7）+ 服务商内海池余量/T2/推荐（H-8）——派单分配辅助。
8. 经营报表三角色内容+下钻（H-15）+ 撤案统计（M-11）——经营复盘。
9. 协调员多对多关联（H-4）+ 服务商催收员佣金（M-13）+ 批次级作废/批次号直达（M-1/M-2）。

**第四批（体验打磨，持续）**
10. 全平台无障碍系统性补强（H-17）+ 列表空态/加载态 + 操作区分组 + 术语统一。
