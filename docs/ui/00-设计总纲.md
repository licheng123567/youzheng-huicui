> 版本：v0.1（UI 设计阶段）
> 依赖：`docs/prd/`（11 模块 70 条故事）、`docs/技术选型决策.md`（Java + 若依 RuoYi = Vue3 + Element Plus 单栈）。
> 本阶段约束：**只做 UI 设计（草图 + 高保真原型），不搭生产骨架、不接后端**。高保真原型产生的是"设计稿性质"的前端原型代码（HTML/CSS/少量 JS + mock 数据），用于演示交互，不等于工程脚手架。

## 1. 设计目标

把 PRD 的 70 条用户故事落成"看得见、点得动"的一套系统，重点验证三件事：

1. **导航主轴成立**：项目 → 批次 → 案件 三层下钻 + 批次号直达，是否顺手。
2. **"一页多角色"复用成立**：同一个案件/列表，平台超管、平台员工、物业负责人、物业协调员、服务商负责人、催收员看到的是**同一套信息架构**，差异只在"能操作什么"和"能看到哪些字段"。
3. **核心闭环走得通**：物业建项目批次 → 平台撮合派单 → 服务商抢单 → 催收员通话催收/标注/承诺 → 协调员法务/存证/结案 → 平台对账。

## 3. 设计原则

1. **复用优先（DRY UI）**：同一业务对象（案件/批次/项目/对账单）全平台只设计**一套**视图组件；不同角色不另起炉灶。差异通过"权限显隐 + 字段脱敏"两层薄壳实现。
2. **单一事实来源（Single Source of Truth）**：案件详情是案件信息的全集页，所有跟进/通话/承诺/法务/存证/状态都挂在这一页的标签页里，不散落多处。
3. **权限驱动显隐**：按钮、菜单、标签页是否出现，由当前账号的权限点决定；无权限的不灰显、直接不渲染（避免泄露功能存在性，呼应 BR-M1-06）。
4. **隔离与脱敏内建**：跨组织数据不可见、结案后服务商/催收员收敛为"只看统计"、物业↔服务商资金零互通——这些不是后端兜底，UI 层先做对（见 `02` 专章）。
5. **导航主轴贯穿**：左侧导航 + 面包屑始终能回到"项目 → 批次 → 案件"；任意列表可凭批次号直达。
6. **工作台聚合待办**：每个角色首页是其待办/提醒聚合（承诺到期、释放预警、最长周期提醒、低余额、工单回执、建议法务、新分配案件等，总纲 §9），按紧急度排序。

```
│ ▸工作台     ├─────────────────────────────────────────────────────────┤
│ ▸撮合派单   │  ┌KPI┐┌KPI┐┌KPI┐┌KPI┐   ← 彩色图标统计卡                 │
│ ▸案件管理   │  └──┘└──┘└──┘└──┘                                       │
│ 财务        │  ┌─卡片：表格 / 详情 / 看板 ───────────────────────────┐  │
│ ▸收佣对账   │  └──────────────────────────────────────────────────┘  │
│ 系统 …      │                                                          │
└────────────┴─────────────────────────────────────────────────────────┘
  深色侧栏 #304156 + 白顶栏 + tagsView + #f0f2f5 内容区（RuoYi-Vue-admin 风格）
```

- **侧栏（深色 #304156）**：菜单按当前角色权限过滤渲染、可分组（业务/财务/系统）；选中项左侧主色竖条 + 蓝渐变高亮 + SVG 图标。
- **顶栏（白）**：左面包屑（项目→批次→案件）；右 **角色切换器（仅原型演示，真实系统按登录账号定）** + 头像账号。
- **tagsView**：访问过的页面成可关闭页签，便于多页切换（RuoYi 标志特征）。
- **面包屑**：始终反映"项目 → 批次 → 案件"路径，每级可点回跳。
- **角色切换器**：原型顶部独有控件，下拉 7 个角色，切换后整页（导航 + 操作按钮 + 字段脱敏）实时变化——这是向用户演示"复用"的核心装置，真实系统不存在此控件。

## 6. "一页多角色"复用机制（核心）

以**案件详情页**为范例，说明复用如何落地。所有列表/详情页同理。

### 6.1 三层结构

```
┌─ 第 1 层：信息全集布局（所有角色相同的骨架）──────────────────┐
│  案件头（业主/房号/应收/状态/归属）                            │
│  ┌ 标签页：概览 | 跟进记录 | 通话&转写 | 承诺 | 工单 | 法务 | 存证 | 操作日志 ┐ │
│  └────────────────────────────────────────────────────────┘ │
├─ 第 2 层：操作区（按权限点显隐按钮）─────────────────────────┤
│  催收员 → [拨号] [标注跟进] [登记承诺] [发催费单] [建议法务] [释放]      │
│  协调员 → [拨号] [标线下回款] [发起法务] [存证] [撤案/坏账结案]          │
│  物业负责人 → [减免审批] [查看对账]   平台超管 → [派单] [代操作] [全量]    │
├─ 第 3 层：字段脱敏过滤（按隔离规则裁剪可见字段）───────────────┤
│  进行中：服务商/催收员可见业主信息+联系方式+跟进+录音               │
│  已结案：服务商/催收员仅见统计、明细全部脱敏（BR-M8-09）             │
│  跨服务商：开放抢单前不可见他商明细；物业不可见平台↔服务商佣金        │
└──────────────────────────────────────────────────────────────┘
```

### 6.2 工程含义（指导后续若依开发，本阶段仅原型体现）

- **一套页面组件 + 一份权限点字典**：每个操作按钮绑定一个权限点 key，渲染时查当前角色是否持有。
- **一份字段可见性策略**：按 `角色 × 案件状态 × 归属关系` 决定字段是否脱敏，集中一处维护。
- 原型用 JS 对象模拟"当前角色 → 权限点集合 + 脱敏规则"，切角色即重渲染，直观演示。

### 6.3 复用清单（同一组件多角色共用）

| 共用视图 | 共用角色 | 差异点 |
|---|---|---|
| 案件详情页 | 全部 6 个 PC 角色 | 操作按钮、脱敏字段、可见标签页 |
| 案件列表 | 全部 | 列字段脱敏、行操作、可见范围（数据隔离） |
| 批次详情 | 平台/物业/服务商 | 物业见收佣比例、服务商见付佣比例（互不可见）、平台全见 |
| 项目档案 | 物业/服务商/平台 | 服务商只读催收依据部分、不见佣金 |
| 对账单 | 平台/物业 或 平台/服务商 | 收佣线 vs 付佣线物理隔离，两套对账互不可见 |
| 工作台 | 全部 | 待办类型按角色不同，框架与组件相同 |

## 8. 状态与反馈规范

| 场景 | 规范 |
|---|---|
| 空态 | 列表无数据给插画 + 一句引导 + 主操作按钮（如"导入批次"） |
| 加载 | 表格骨架屏 / 区域 loading |
| 二次确认 | 不可逆动作（撤案/坏账/作废/释放/停用）弹确认框，须选原因的强制选原因 |
| 留痕提示 | 代操作、派单、抢单、释放、结案等成功后 toast 提示"已记录操作日志" |
| 权限缺失 | 无权限功能**不渲染**（非灰显），直链访问返回"无权限"且不泄露对象存在 |
| 风险/隔离 | 脱敏字段显示为"***（已脱敏）"或整块隐藏，并在角色切换器旁标注当前可见性策略（仅原型） |
