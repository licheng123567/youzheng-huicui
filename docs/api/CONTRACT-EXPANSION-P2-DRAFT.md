## 子片 1：个人中心
**PRD 现状**：密码/手机为**管理员重置/改绑**模型（负责人交接 BR-M1-02/12、US-M1-09；已有 `POST /members/{id}/reset-password`，member.manage）。**无自助改密定义。**

### 1a 个人资料查看（PRD 已有数据 · 无契约改动）
- ProfileView 页：展示 GET /me（账号/姓名/角色/组织/权限点/数据范围）+ 退出。纯前端。

### 1b 自助改密码（⚠ 新产品决策）
- 若采纳：`+POST /me/password { oldPassword, newPassword }`（self·x-permission:none·校验旧密码，错误旧密码 401，弱密码 422）
- 需补 BR（自助改密：任一登录账号可改本人密码，校验旧密码，与管理员重置并存）
- 不采纳：个人中心仅资料查看，改密仍走管理员重置

### 1c 自助改手机/资料
- **不开**：PRD 手机=负责人交接（管理员改绑），自助改手机会与交接机制冲突 → 维持管理员侧

## 子片 2：全局搜索
**PRD 现状**：BR-M4-22 定义**通话记录搜索**（业主姓名/房号/电话 + 项目/批次/时间，按数据范围裁剪）；**无「全局跨实体」定义。**

### 方案 A（PRD 对齐 · 窄 · 推荐）
- `+GET /search?q=&type=case` → 案件命中（业主姓名/房号/户号/电话模糊），x-data-scope:range（按角色裁剪），脱敏遵循 BR-M3-21a
- 直接落地 BR-M4-22 的搜索字段，范围明确、风险低

### 方案 B（扩展 · 宽）
- `+GET /search?q=` → 分组返回 案件/批次/项目 命中（SearchResult{cases[],batches[],projects[]}）
- 超出 PRD 当前定义，属增量功能（需补"全局搜索"BR）

### 前端
- AppLayout 顶部搜索框 → 结果页/下拉；命中案件直达作业台
