## 二、到期执行方式（关键决策——影响是否建调度器）
设计上 T1/T2/TC 到期是**自动执行**（BR-M3-02/08/13 均写"自动"），但当前**无调度器**，deadline 只存不执行。两种范围：

- **方案 A（完整·建调度器）**：加 `@Scheduled` 周期任务（如每 5 分钟）扫描到期案件 → 事务内执行转移（T1→平台公海 / T2→平台公海 / TC→释放），幂等 + 留痕(activity + 事件日志) + 守护栏①(BR-M3-16)。同时补 T2/T1 预警。**真正闭合公海自动流转**。
- **方案 B（仅预警·不自动执行）**：本轮只补 **T2 预警(VL)+T1 预警(平台)** 进 workbench/消息中心 + 公海倒计时展示；到期转移仍靠**平台/服务商手动操作**（现有再派/退回/释放端点）。**不建调度器**，规避定时任务的运维/并发复杂度。
