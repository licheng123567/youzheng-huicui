## 审计（opus 只读审计替代 codex —— codex 在本环境网络不可达）

**评分 86/100 · 无 BLOCKER · 有条件可冻结 v1.1。**

护城河（资金双线隔离 / 佣金 comm=base×rate 分数 / 状态机 CAS / 双线裁剪 / public 最小披露）实现严谨，无可利用的越权、跨租户泄露或 SQL 注入。子代理多个"BLOCKER"（verify 泄露 / JSONB 注入 / LIKE-ORDER BY 注入 / CloseM8 status=kind）经复核为误报。

### 冻结前已修（本次）
- **H-1** `JwtAuthFilter` `endsWith("/verify")` 过宽 → 改精确 `/evidence/[^/]+/verify`（防未来 /verify 端点被静默免鉴权）。
- **H-2** 幂等拦截器首次失败后同键无法重试 → 改为**仅 2xx 成功后登记键**（afterCompletion），失败可安全重试。
- **H-4** 减免额可超应收致 `reduce_after_cents` 负（传导业主账单/对账）→ 加 `0 ≤ 减免 ≤ 应收` 校验。
- **H-5** 结清/反结清基于非锁快照无状态 CAS → UPDATE 加 `status IN(...)` 前置态条件。
- **M-1** `handleOther` 把一切异常吞成 422 掩盖真实 bug → 兜底前 `log.error` 记完整堆栈（可观测性）。

### 已知债结论
- **re-claim 200→409**：判定为**按幂等设计正确，非缺陷**（同一持有人重复抢自己已持有的案应 200；他人持有抢则 409 `BIZ_ALREADY_CLAIMED`）。债务关闭。

### fast-follow（v1.1 后补丁窗口，不阻断核心闭环）
- H-3 `nextSeq` 并发重号唯一冲突落 422 应映射 409；M-2 佣金线审计落库；M-3 me/settlement 按 provider 过滤；M-5 PaymentRequest documentUrl/sealed 占位确认。
